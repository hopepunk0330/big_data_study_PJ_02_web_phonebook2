### 2026-07-14 — PRD v1.0→v1.1 개정 브리프 (사용자 승인된 정합화 작업)
- planning-pl 요청: PRD를 PDF(v1.0)에서 md(v1.1)로 최초 전환하며 화면정의서 v1.3의 SCR-004(비밀번호 찾기)를 PRD에 정식 반영. PRD v1.0 PDF 전체를 직접 재확인했다(G1~G6, N1~N5, UC-01~08, PR-01~12, AC-01~17 전부 실제 존재 확인).
- 확정 판단 4건을 service-concept.md §3-1에 정리: (1) AC-04를 "로그인 실패 응답(POST /auth/login)에 한정"으로 범위 명시하는 문구로 개정(삭제·완화 아님), (2) N1을 분리해 "이메일 인증"만 남기고 "비밀번호 찾기/변경"은 비목표에서 제거, (3) §11 표의 "비밀번호 변경/찾기" 행을 "본인 확인 강화(이메일/보안질문, 향후 확장)"로 갱신, (4) UC-09/PR-13/AC-18 신설 문구 확정.
- 번호 검증 결과: UC-09/PR-13/AC-18 전부 PRD 원문 기준 실제 빈 번호 맞음. PRD는 원문에 "FR-" 인용 관례가 없음(§10처럼 문서 단위로만 참조) — 따라서 FR-14/15(tech-architect가 01 문서에 반영 예정)를 PRD에 직접 인용하지 말고 "01 문서 참조"로만 남기도록 권고.
- PR-13의 목표(G) 매핑은 G1으로 제안(비밀번호 찾기가 로그인 가능 상태로 복귀시키는 기능이라는 논리) — G7 신설도 가능하나 이번 브리프 범위 밖이라 언급만 하고 G1 편입을 기본안으로 제시.
- service-concept.md §4(구 "미해결" 절)를 갱신해, PRD 쪽은 이번에 해결되지만 01/03 문서와 화면정의서의 "FR-13" 임시 표기는 후속 라운드 필요함을 명시.

### 2026-07-14 — PRD v1.1→v1.2 개정 브리프: §4/§7 사용자 여정 다이어그램에 비밀번호 찾기 흐름 반영 (사용자가 v1.1 라운드의 "안 넣는다" 결정을 뒤집고 승인)
- planning-pl 요청: PRD §4(제품 구성, 정적 구조도)와 §7(전체 워크플로우, 사용자 여정)에 비밀번호 찾기 흐름을 추가. PRD v1.1·화면정의서 v1.4(SCR-004 Mermaid) 원문을 직접 재확인했다.
- 확정 판단: (1) §4는 "로그인 이후 데이터 격리"만 보여주는 정적 구조도라는 이유로 mermaid 코드는 그대로 두고 본문에 한 줄 각주만 추가(최종 권고, 근거 명시), (2) §7은 U2(로그인) 노드에서 분기하는 요약 노드 PW(노드 텍스트·화살표 레이블까지 확정: "비밀번호를 잊음" → PW → "재설정 완료 → 로그인 화면 복귀" → U2) 1개 + 화살표 2개만 추가, SCR-004가 완료 후 SCR-001(로그인 화면)로 복귀하는 지점과 논리적으로 같은 U2로 재합류시키고 실패 케이스(404)는 그리지 않기로 확정(SCR-004 전체 상태 분기 복제 방지), (3) 기존 v1.1 "다이어그램 변환 안내" 블록은 지우지 않고 그 아래에 v1.2 블록을 추가하는 정확한 문구 제시(§4/§7/PRD 최상단 변경 이력 3곳), (4) 01/03/tech-architecture.md/화면정의서 점검 결과 새 ID 신설·추가 갱신 불필요 확인(오히려 화면정의서 §3/SCR-001의 기존 분기 구조와 정합성이 개선됨) — 단 향후 SCR-004 번호가 바뀌면 PW 노드 텍스트도 함께 갱신해야 한다는 유지보수 주의사항만 기록.
- service-concept.md §3-1 (5)(원래 "안 넣는다" 결정)는 삭제하지 않고 상단에 "갱신됨 — §3-2 참고" 표시만 추가, 새 §3-2에 이번 판단 전체를 confirmed 브리프로 정리.
