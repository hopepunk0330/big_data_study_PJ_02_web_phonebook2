- 커버됨: 로그인 폼, 로그인 401 실패 배너(login-알림창, "SCR-900 Message Banner"), 회원가입 폼(Join), 메인 목록+카테고리 사이드바(main, 카테고리 CRUD UI 포함), 연락처 수정(main-수정, 모달 방식), 연락처 삭제 확인(main-삭제, 모달 "연락처 삭제" 전용), 검색결과 없음(main-검색없음, EmptyState), 성공 토스트(main-알림창, "추가되었습니다").
- 누락 발견(design-pl 3-B 전체확장 단계용 목록, 우선순위 없이):
  1. 회원가입 실패 상태(409 아이디중복/422 형식위반) — 02문서 §4 SCR-001, PRD AC-02.
  2. 회원가입 성공 안내("가입 완료! 로그인해 주세요") — 02문서 §4 SCR-001.
  3. 메인 화면 오류 배너(전화번호중복 409/형식위반 422/카테고리없음 404/카테고리이름중복 409) — main-알림창은 성공 전용, SCR-900은 "모든 화면"에 적용.
  4. 카테고리 삭제 확인 모달 — main-삭제는 "연락처 삭제" 전용으로 확인, 카테고리용 별도 없음. PRD §5 UC-07, §8.
  5. 카테고리 사용중 삭제거부(409, "N건 있어 삭제할 수 없습니다") 안내 상태 — PRD가 "2차 과제 대표 엣지케이스"로 명시, 프레임 없음.
- 경계 케이스: (a) 연락처 수정이 문서는 "인라인 폼 편집"인데 확정 디자인은 모달 — 이미 확정된 패턴이라 판단 대상 아니나 문서와 불일치 기록만 남김. (b) 카테고리 이름수정은 문서상 브라우저 native prompt() — 커스텀 모달 필요 여부 미정. (c) main-검색없음의 카피가 "검색 트리거" 전용이라 신규가입 직후 "진짜 빈 목록" 상태에 그대로 못 씀. (d) login/login-알림창에 PRD 비목표(N1)인 "비밀번호 찾기" 버튼이 존재(Join엔 숨김) — 범위 외 기능 노출.
- 접근성 메모: 사이드바 카테고리 수정/삭제 아이콘버튼 28×22px, 목록 행 수정/삭제 41×25px — WCAG 2.1 AA 권장 타깃 크기(24×24 이상, 이상적 44×44)에 못 미쳐 오탭 위험. 신규로 만들 오류/성공 배너는 기존처럼 아이콘+텍스트 병기 패턴 유지 권장(색상 단독 구분 금지).
