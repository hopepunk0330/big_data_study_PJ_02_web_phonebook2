이 파일은 planning-writer의 최근 작업 로그(휘발성, 5개 캡)만 남깁니다. 각 기획 문서의 확정 내용 자체는 여기 없습니다 — `docs/planning/*.md`(현재 버전)와 `docs/planning/old/`(과거 버전 실물)가 소스입니다. 컨셉/BM 판단은 `docs/planning/service-concept.md`(service-planner 관리), 기술 판단은 `docs/planning/tech-architecture.md`(tech-architect 관리)를 따릅니다.

## 작업 로그

### 2026-07-14 — PRD v1.1 → v1.2 (§4/§7 사용자 여정 다이어그램에 비밀번호 찾기 반영)
- 사용자가 §3-1 (5) 결정을 뒤집어 요청한 라운드. 근거: `service-concept.md` §3-2(service-planner 브리프, 신설 절 — mermaid 코드까지 그대로 확정해 둠).
- §4(정적 구조도): mermaid `flowchart LR` 자체는 한 글자도 안 건드림, 코드 블록 밖 본문에 각주 한 줄만 추가("로그인 실패·비밀번호 분실 복구 경로는 이 구조도 범위 아님 — §7 참고").
- §7(사용자 여정): 노드 PW 1개(U2 다음 줄) + 화살표 2개(U2→PW, PW→U2, 기존 `V-->U1-->U2-->M` 줄 바로 아래) + `class V,U1,U2,LO,R,PW step`(기존 classDef 재사용, 새 클래스 안 만듦) 삽입. SCR-004의 404/422 등 세부 분기는 복제하지 않고 PW 노드 텍스트에 "상세: 화면정의서 SCR-004 참고"로 위임. 재합류 지점은 M이 아니라 U2(SCR-004 완료 시 SCR-001 로그인 화면 복귀와 논리적으로 동일 지점).
- 기존 "다이어그램 변환 안내(v1.1)" 블록은 지우지 않고 그 아래에 "다이어그램 갱신 안내(v1.2)" 블록을 §4/§7 각각 추가(이력 보존).
- 상단 표 버전 v1.1→v1.2, 변경 이력 최상단에 v1.2 항목 추가(브리프 문구 그대로 사용), 기존 v1.0/v1.1 이력 항목은 유지.
- `diff` old-v1.1 vs 신규 v1.2 결과, 위에 명시한 지점(버전 값 1줄, 변경이력 1줄, §4 각주+안내블록 3줄, §7 노드+화살표2+class수정+안내블록 5줄) 외 100% 동일 확인 — §1/§2(N1 각주 제외)/§3/§5/§6/§8~§11 전혀 안 건드림.
- mermaid-cli(npx @mermaid-js/mermaid-cli)로 §4/§7/§8 다이어그램 3개 전부 파싱·렌더 검증 완료(§8은 이번에 안 건드렸지만 회귀 확인 겸 포함).
- 구 v1.1 파일은 `docs/planning/old/04_연락처관리_웹서비스_PRD_v1.1.md`로 이동 보존.

### 2026-07-14 — 화면정의서 v1.2 → v1.3 (Mermaid flowchart 보완)
- `docs/planning/02_연락처관리_웹서비스_화면정의서_v1.2.md`를 v1.3으로 올림. v1.2는 팀 표준 4종 세트(목적/와이어프레임/구성요소-동작 표/Mermaid 플로우차트) 중 Mermaid만 5개 화면(SCR-001, SCR-004, SCR-002, SCR-003, SCR-900) 전부 빠져 있던 것을 보완.
- surgical 준수: 각 화면 절의 "구성 요소와 동작" 표 바로 다음에 "**화면 흐름 (Flowchart)**" 소제목 + mermaid 코드블록만 삽입, 기존 프로즈는 한 글자도 안 건드림. 삽입한 블록만 정규식으로 걷어내 v1.2와 diff한 결과 버전 번호·변경이력 추가 줄 외 100% 동일함을 확인(검증 완료).
- 근거: service-planner가 준 상태 분기 브리프(성공/실패/조건/화면전환)를 그대로 mermaid 문법으로 옮김 — 내용 판단은 하지 않음. SCR-002/003은 "401이면 다른 실패보다 우선해 SCR-001 강제전환"하는 공통 규칙을 모든 분기가 공유 노드(Z)로 수렴하는 구조로 표현.
- 처음에 SCR-002/003 플로우차트 뒤에 "공통 규칙" 설명 blockquote를 추가로 넣었다가, 지침이 "flowchart 코드블록만 삽입"이라 명시한 것을 뒤늦게 인지하고 제거함(다음엔 처음부터 코드블록 외 텍스트 추가 안 하도록 주의).
- mermaid-cli(npx @mermaid-js/mermaid-cli)로 5개 다이어그램 모두 파싱·렌더 검증 완료(문법 오류 없음).
