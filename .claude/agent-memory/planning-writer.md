### 2026-07-15 — TRD v1.0 → v1.1 (static/ 물리적 위치 정정: backend/·static/·frontend/ 형제 폴더화)
- 근거: `docs/planning/tech-architecture.md` §1-1/§5(tech-architect 브리프, 신설 절 — canonical 트리 구조와 결정 이력 이미 반영 완료), 브리프 파일(`trd-structure-brief.md`) §1~6의 "그대로 교체" 텍스트를 그대로 옮김 — 이번 라운드는 판단 없이 문서화만.
- **정정 핵심**: v1.0은 "같은 오리진 유지 → CORS 불필요"라는 옳은 결론의 근거로 "물리적 폴더 중첩 필수"라는 틀린 전제를 썼음(`static/`을 `backend/` 아래 중첩). FastAPI `StaticFiles(directory=...)`가 파일시스템 경로와 무관하게 서빙 프로세스 기준으로 오리진을 결정한다는 점을 근거로, `static/`을 `backend/`·`frontend/`와 형제인 프로젝트 루트 폴더로 재배치.
- 4곳 교체: ① §2 mermaid subgraph 라벨(구조는 불변, 텍스트만) + CORS 문단 전면 재작성, ② §3 전체 교체(트리+마운트 코드+근거 문단, 기존 "backend/CLAUDE.md에 명시된..." 문장은 새 본문 마지막 문단으로 흡수·중복 제거), ③ §4-4(브리프가 "§7"이라고 오기했으나 실제 위치는 §4-4임을 원문 대조로 확인 후 정정 반영) NFR-05 문단에 "backend/ 바깥" 명확화 문구 추가, ④ §9-1 docker-compose 주석("backend/ 기준"→"프로젝트 루트 기준") + 실행 위치 안내 문장 신설.
- §11 FR-13 행은 브리프 판단대로 변경 없음(원래도 `static/index.html`만 표기해 정정 전부터 옳았음). §1/§4-1~4-3/§5/§6/§8/§10/§12는 전혀 손대지 않음.
- 변경 이력 블록이 이 문서에 처음 신설됨(v1.0에는 changelog 절이 없었음) — 03 문서 포맷(헤더 표 아래 `> **변경 이력**` blockquote, 최신이 맨 위)을 그대로 참고.
- `diff`(old v1.0 vs 신규 v1.1)로 변경 지점이 브리프 지시 위치와 정확히 일치함을 확인 — 버전값 1줄, changelog 4줄, §2 mermaid 라벨 4줄+CORS 문단 1줄, §3 전체(트리+마운트 코드+근거, 다수 줄), §4-4 문장 1줄, §9-1 주석 1줄+안내문장 1줄 외 100% 동일.
- mermaid-cli(npx @mermaid-js/mermaid-cli)로 문서 내 mermaid 코드블록 4개(§2 flowchart, §4-1 erDiagram, §5-1 sequenceDiagram, §7 flowchart) 전부 파싱·렌더 검증 완료.
