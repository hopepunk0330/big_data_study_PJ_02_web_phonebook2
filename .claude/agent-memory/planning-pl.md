### 2026-07-14 — "비밀번호 찾기" → "비밀번호 재설정" 명칭 통일 라운드: 완료 (01/02/03/04 전체)
- 배경: 사용자가 이 흐름의 이름을 "비밀번호 재설정"으로 통일하기로 확정(범위: 기획서 전체 + Figma). Figma의 Link 컴포넌트 텍스트는 사용자가 이미 직접 바꿔놨고, 이 팀은 문서만 담당. 이번엔 새 번호 배정 없이 명칭/서술만 바뀌는 작업이라 tech-architect는 부르지 않고 service-planner만 라우팅(내 판단: 흐름명 vs 1단계 서술 표현은 아키텍처가 아니라 컨셉 판단).

API 경로/스키마 클래스명(`find-password`, `FindPasswordIn/Out`)은 코드 식별자라 이번 범위 밖(tech-architect 소관)으로 명시. `service-concept.md` §3 본문을 현재 명칭으로 갱신 + §3-1/§3-2는 감사 기록이라 옛 명칭 그대로 보존(각주만 추가) + 신규 §3-3에 4개 문서 위치별 실행 브리프 작성 + §5 결정 이력 추가.

Mermaid 코드블록 13개 전부 mermaid-cli 재검증. 구버전 4개 `docs/planning/old/`로 이동.
