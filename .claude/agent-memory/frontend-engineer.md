**B. 구체 버그 5건**:
1. **로그인 카드 gap 불균일 — 근본 원인은 order 트릭과 `* + *` 형제결합자의 충돌**: `.auth-card form > * + * { margin-top:12px }`는 DOM 순서 기준인데 login-form은 CSS `order`로 시각순서만 맞춰놔서(비밀번호를 DOM상 먼저 두는 트릭), 실제 계산된 간격이 12~32px로 제각각이었다(divider→회원가입 버튼만 28px로 유독 넓었던 것도 이 때문). `get_design_context(936:1191)` 재실측 결과 Content는 `flex-col gap-[20px]` — `* + *` 마진 트릭을 걷어내고 `.auth-card form`에 `gap:20px`만 적용(자식 개별 margin은 0으로 죽임)해 order 트릭과도 안전하게 호환되는 균등 20px로 교체. 여기서 그치지 않고 카드 전체(로고→타이틀→필드→ConfettiFooter)도 브랜드로우 자체 margin-bottom(16, layout.css 공용값)과 컨페티푸터 margin-top(12) 등 남은 하드코딩 마진이 똑같이 불균일을 만들고 있어 `.card-body` 자체를 `flex-col gap:20`으로 바꾸고 그 안의 개별 마진들(brand-row/auth-card-title/confetti-footer)을 0으로 정리 — 카드 전체가 Figma의 Content 컨테이너와 동일한 단일 gap 규칙을 따르게 통일했다.
2. **CategoryNav/카테고리 관리 목록 스크롤**: `.nav-list`에 `max-height:238px`(937:298 실측 고정값) + `overflow-y:auto` 추가. `.manage-list`는 Figma에 고정값이 없어 "균형 잡힌 값 판단" 지시였는데, design-system.md 38-1절에 이미 실측 기록된 "카테고리 4개 기본 상태 CatRows 높이 114px"를 근거로 재사용(임의 수치 발명 안 함) — `max-height:114px`+`overflow-y:auto`. Playwright로 카테고리 9개 추가해 실제 scrollHeight(464/226) > clientHeight(238/112)로 스크롤 동작 확인, `.sidebar` 자체 높이(576)가 `.app-shell`(580)을 벗어나지 않음도 확인.
3. **로그인 화면 FOUC**: `#screen-auth`에 `hidden` 기본값이 없어 `/auth/me` 응답 전까지 로그인 화면이 잠깐 보였다 사라졌다 — `hidden` 추가, `init()`이 응답 후 `showLogin()`/`showMain()` 중 하나를 호출해야만 보이게 정정. `test_auth.py`는 `page.goto`후 로케이터 auto-wait라 회귀 없음 확인.
4. **Focus 링 브라우저 기본 아웃라인 미제거**: `.btn`/`.row-action`/`.row-action-icon`/`.input`/`.input-corner` 5개 선택자에 `:focus-visible`과 별개로 `:focus { outline:none }`을 추가(기존엔 `:focus-visible` 안에만 있어 마우스 클릭 시 브라우저 기본 아웃라인이 남아있었음). Playwright로 **실제 마우스 클릭 스크린샷**(`audit-focus-01`, 로그인 버튼 — 링 없음 확인)과 **Tab 키 스크린샷**(`audit-focus-02`, 비밀번호재설정 링크 — 커스텀 검정 링 확인)을 따로 찍어 육안 대조 완료. 참고로 이 라운드 도중 layout.css에 `*:focus{outline:none}`/`*:focus-visible{box-shadow:...}` 전역 규칙이 추가로 반영됨(사용자/린터, 더 포괄적인 안전망) — 내 컴포넌트별 규칙과 값이 동일해 충돌 없음, 중복이지만 무해해 그대로 둠.
5. **로그인 "or" 구분선 관련 — 4번과 통합 처리**: 1번 항목의 gap:20 정리로 이미 해소.

## 2026-07-19 — 긴급 크로스브라우저 버그: 연락처 테이블 sticky-thead를 WebKit-safe 별도 div 헤더로 재구현 (dev-pl 지시, WebKit 실측 필수)

**배경**: 직전 라운드(2026-07-18)에서 만든 `.contact-table thead th { position:sticky }` 방식이 Chromium은 완벽했지만, 사용자 실제 브라우저(Safari/WebKit 추정)에서 스크롤 시 헤더가 본문과 같이 움직인다는 반복 재현 보고 — `playwright install webkit`으로 실제 WebKit 엔진을 설치해 원인부터 확정하고, 확정되면 헤더를 테이블 밖 별도 div로 완전히 분리하라는 지시.

**1) 원인 재현 시도 — 정직하게 보고할 결과: 번들 WebKit(2311, WebKit 26.5)에서는 재현 안 됨**: 연락처 12건 시드 후 실제 마우스 휠(`page.mouse.wheel`)로 6단계 스크롤하며 매 단계 헤더 bbox를 찍었는데, `position:sticky`가 모든 단계에서 완벽히 고정(`getComputedStyle().position === 'sticky'`, y좌표 불변) — 이 프로젝트의 CSS가 `border-collapse:separate`를 이미 쓰고 있어(빈 상태 테두리 버그 회피용으로 이전에 결정됨) 정확히 "sticky+collapse:collapse 조합에서만 깨지는" 유명한 WebKit 버그 조건을 안 건드리고 있었다는 게 원인으로 추정된다. 즉 **번들 Playwright WebKit(최신 버전)로는 사용자가 보고한 증상을 재현하지 못했다** — 실제 사용자 Safari가 더 오래된 버전이거나(해당 WebKit 버그는 과거 여러 해에 걸쳐 부분적으로만 고쳐짐), 헤드리스 테스트가 못 잡는 실제 환경 차이일 가능성이 높다는 점을 dev-pl에 정확히 보고. 이 불확실성에도 불구하고, sticky-th on table 자체가 여러 WebKit/Safari 버전에 걸쳐 산발적으로 문제를 일으켜온 기법이라(브라우저 버전 의존적) dev-pl 지시대로 **테이블 밖 별도 div 헤더**로 교체하는 것이 근본적으로 더 안전하다고 판단해 예정대로 진행.

**2) 재구현 — 헤더를 `<table>` 밖 별도 `.contact-table-header` div로 분리**: `.contact-table-wrap`(신규 최상위 래퍼, border/border-radius/overflow:hidden을 여기로 이동)이 `.contact-table-header`(고정, `role="row"`+`.cth-cell[role=columnheader]` 5개, sticky 없이 그냥 스크롤 영역 밖 static 배치라 어떤 브라우저에서도 깨질 여지가 원천적으로 없음)와 `.contact-table-scroll`(스크롤 영역, `<table>`은 `<thead>` 없이 `<tbody>`만)을 감싼다. 컬럼 폭은 `.contact-table-wrap`에 `--col-name-width`(80)/`--col-phone-width`(128)/`--col-type-width`(80)/`--col-actions-width`(110) 4개 CSS 변수로 선언해 헤더 div의 `.cth-*` 셀과 tbody `td:nth-child` 규칙이 그대로 같은 변수를 참조 — 값이 바뀌어도 두 쪽이 항상 동기화된다. `aria-label="연락처 목록"`을 스크롤 테이블에 남겨 접근성 컨텍스트 유지.

**3) 재구현 도중 발견한 진짜 회귀 버그(진지하게 다뤄야 했던 부분) — `td.actions`의 `display:flex`가 `table-layout:fixed`의 컬럼 폭 분배를 깨뜨림**: `<thead>`를 제거하고 나니 "첫 번째 행"이 이제 tbody의 실제 데이터 행이 되는데, 그 행의 마지막 `<td class="actions">`에 기존부터 있던 `display:flex`가 걸려 있으면 Chromium/WebKit **둘 다** table-layout:fixed의 auto 컬럼(주소) 폭 계산이 완전히 틀어진다(실측: 주소 컬럼이 remaining space의 약 65%만 받고 테이블 우측에 133px 빈 공간이 남음 — 격리된 최소 재현 HTML로 원인을 좁혀 확인, `<thead>` 유무와 무관하게 flex-display td가 fixed-table의 첫 행에 있으면 항상 발생). **수정**: `td.actions` 자체는 순수 `table-cell`로 되돌리고(`padding:0`), 버튼 2개를 새 `.actions-inner`(`display:flex`, 자체 padding 10px 16px) div로 감싸는 방식으로 flex를 셀 안쪽 래퍼로 옮겼다 — 최소 재현 HTML/실제 페이지 양쪽에서 이 변경만으로 컬럼 폭이 정확히 원상 복구됨을 확인. `static/app.js`의 `renderContactRows()`에서 두 버튼을 `<div class="actions-inner">`로 감싸도록 수정.

**4) 스크롤바 거터 동기화**: 헤더가 스크롤 영역 밖으로 나오면서, 세로 스크롤바가 나타날 때(9행 이상) 스크롤 컨테이너의 실제 컨텐츠 폭이 스크롤바 두께만큼 줄어들 수 있어 헤더와 어긋날 위험이 있다 — `syncContactTableHeaderGutter()`(신규, `app.js`)가 `scrollEl.offsetWidth - scrollEl.clientWidth`를 실측해 헤더에 `padding-right`로 되돌려주는 방식으로 대응(렌더 직후 + `window resize`마다 호출). 실측 결과 이 프로젝트의 `::-webkit-scrollbar`/`scrollbar-width:thin` 스타일은 오버레이형이라 두 엔진 모두 거터가 0이었지만, 다른 브라우저/OS 스크롤바 설정(항상 공간 차지하는 클래식 스크롤바)에서도 안전하도록 값을 하드코딩하지 않고 실측 로직을 남겨뒀다.

**변경 파일**: `static/index.html`(`.contact-table-scroll`을 감싸는 `.contact-table-wrap` 신규 + `.contact-table-header` div 신규, `<thead>` 제거, `<table>`에 `aria-label` 추가), `static/app.js`(`renderContactRows`의 액션 버튼을 `.actions-inner`로 래핑, `syncContactTableHeaderGutter()` 신규 + 렌더 후/resize 호출), `static/css/layout.css`(`.contact-table thead th` sticky 규칙 제거, `.contact-table-wrap`/`.contact-table-header`/`.cth-*` 신규, `td.actions`→`.actions-inner`로 flex 이동, 컬럼 폭 CSS 변수화, max-height 404→368px로 재계산(헤더가 더 이상 스크롤 영역 안에 없으므로)).

**검증**: WebKit으로 12건 시드 후 마우스 휠 스크롤 전/후 헤더 y좌표·컬럼(헤더 5개 vs tbody 5개) x좌표를 매번 비교하는 스크립트로 **헤더 고정 True, 스크롤 전/후 컬럼 정렬 True**(짧은 주소/매우 긴 주소 혼재, ellipsis 정상 트리거 포함) 확인, Chromium도 동일 스크립트로 회귀 없음 재확인. `python -m pytest tests/test_auth.py tests/test_contacts.py tests/test_categories.py -q` → **118 passed**, `tests/test_persistence.py` → **4 passed**(총 122, 회귀 없음 — `.actions`/`thead` 마크업 변경이 기존 로케이터에 영향 없음, 테스트가 전부 placeholder/역할 기반이라 안전). 스크린샷: `contacts-table-scroll-webkit-BEFORE-01/02`(수정 전 WebKit, 재현 안 됨 확인용), `contacts-table-scroll-webkit-AFTER-01/02`·`contacts-table-scroll-chromium-AFTER-01/02`(재구현 후 스크롤 전/후, 양쪽 엔진). **"완료" 선언 안 함 — WebKit 번들 버전에서 원인 재현에 실패했다는 점을 반드시 dev-pl에 전달, 실제 사용자 브라우저로 재검증 필요.**
