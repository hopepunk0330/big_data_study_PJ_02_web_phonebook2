### 2026-07-12 — 파일럿 3화면 Alert/Button 상속 진단 (수정 없음 — 전부 자동 상속 확인)
- 배경: design-systems가 컴포넌트 레벨에서 Alert(`104:2`, `104:108`)에 `Elevation/Raised` Effect Style 바인딩 추가, Button(`97:8`, `97:47`)의 `Style=Primary, Content=Text/IconText, State=Default` 2개 variant 텍스트를 `component/button-text-disabled`(대비 1.22:1)에서 `color/text-primary`(대비 5.31:1)로 재바인딩. 파일럿 3화면(Login `108:2`/Contacts — With Data `109:2`/Contacts — Empty `111:2`)에 배치된 인스턴스가 개별 오버라이드로 상속이 끊겼는지 진단하는 것이 목적 — 화면 재작업이나 컴포넌트 조립이 아님.
- 방법: 화면 재디자인 없이, `use_figma` 읽기 전용 스크립트로 각 화면의 Alert/Button 인스턴스 노드를 직접 조회해 `instance.overrides`(오버라이드된 필드 목록)와 top-level fill/effect의 `boundVariables`, Label/Message 텍스트 child의 fill `boundVariables`를 확인. 리졸브한 변수 ID를 `getVariableByIdAsync`로 이름까지 재확인(예: `VariableID:95:19` = `color/text-primary`, `VariableID:114:2/5/7/9` = elevation shadow 토큰).
- **Login (`108:2`)**: Button 2개(로그인 `108:125`=Primary+Text+Default, 회원가입 `108:127`=Secondary), Alert 1개(`108:130`=Type=Error). 셋 다 `overrides`에 `height/width/primaryAxisSizingMode`(레이아웃 사이징, 이 화면 조립 시 의도적으로 지정한 값)만 있고 `fills`/`effects`는 오버라이드 없음 → 로그인 버튼 Label은 `color/text-primary`(#1C1F21) 자동 상속 확인, Alert는 `Elevation/Raised` 그림자 토큰 자동 상속 확인. **오버라이드 없음, 자동 상속 확인 — 수정 불필요.** 스크린샷으로 로그인 버튼 텍스트 고대비 가독성과 Alert(Error) 카드의 미묘한 그림자 경계 육안 확인 완료.
- **Contacts — With Data (`109:2`)**: Button 5개(로그아웃 `109:13`=Secondary, 카테고리 추가 `110:58`=Primary, 연락처 추가 `110:192`=Primary, 검색 `110:197`=Primary, 전체 `110:199`=Secondary), Alert 1개(SCR-900 Success 데모 `110:205`=Type=Success). 전부 `fills`/`effects` 오버라이드 없음, Primary 3개 모두 Label이 `color/text-primary` 자동 상속, Alert도 `Elevation/Raised` 자동 상속. Secondary 2개(로그아웃/전체)는 원래도 정상이라 범위 밖이지만 진단만 수행 — 이상 없음. **오버라이드 없음, 자동 상속 확인 — 수정 불필요.** 스크린샷으로 Success Alert 그림자 경계 확인.
- **Contacts — Empty (`111:2`)**: Button 5개(로그아웃 `111:13`=Secondary, 카테고리 추가 `111:315`=Primary, 연락처 추가 `111:331`=Primary, 검색 `111:336`=Primary, 전체 `111:338`=Secondary) 전부 `fills` 오버라이드 없음, Primary Label은 `color/text-primary` 자동 상속. **Alert 인스턴스는 이 화면에 애초에 존재하지 않음**(`findAll`로 전체 화면 스캔해 확인 — 배경 문서의 "SCR-900 메시지 배너" 예상은 이 화면엔 해당 안 됨, 빈 상태라 표시할 성공/오류 메시지가 없는 게 정상). **오버라이드 없음, 자동 상속 확인 — 수정 불필요.**
- 구조 미변경: 이번 라운드는 전부 읽기 전용 스크립트만 실행(노드 생성/수정 0건)이므로 진단에 사용한 `get_metadata` 결과 자체가 곧 현재 상태 — 별도 전/후 비교 불필요, 3화면 모두 자식 수·위치 변경 없음.

- 하지 않은 것: 노드 생성/수정/오버라이드 제거 없음(할 필요가 없었음), Secondary/Disabled variant 미수정(범위 밖, 이상도 발견 안 됨), 레이아웃/텍스트/배치 변경 없음, design-qa 투입 등 다음 단계 임의 진행 없음.
