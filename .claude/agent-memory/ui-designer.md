### 2026-07-13 — SCREENS 결함 수정 라운드 3 (Row Action Button 텍스트 복원 + Search Icon/추가버튼/Select 화살표, 인스턴스 한정)
- 배경: design-pl 실행 브리프로 파일럿 3개 중 Login(`153:19`, 무수정 대상 외)/Contacts — With Data(`153:373`)/Contacts — Empty(`153:547`) 인스턴스에 국한된 결함 4건 수정. 마스터 컴포넌트(Row Action Button `166:421`/Input `100:46`/Button `97:47`/Select `101:64`) 자체는 미수정, design-systems·design-qa 미투입.
- **결함1 — Row Action Button 텍스트 복원**: 대상 실측 결과 Table Row(With Data, 6행×2=12곳) + 사이드바 Cat Actions(2화면×4행×2=16곳) 총 28개 인스턴스. 전부 `detachInstance()`(형태 자체가 아이콘→텍스트로 바뀌므로 오버라이드 불가, 브리프 지시대로 detach). **API 함정 발견**: 나선 인스턴스(edit/delete-action)가 INSTANCE 타입 부모(Table Row 자체) 안에 있을 때 그 나선 인스턴스를 detach하면 부모 Table Row 자체가 암묵적으로 detach되며 ID가 통째로 바뀜(sibling까지 재발급) — 이 라운드에서 처음 실측 확인. 대응: row를 y좌표가 아니라 "인덱스"로 매 반복마다 `table.children.filter(name==='Table Row')[i]`로 재조회(첫 실패 원인은 y좌표 캐시 — action 버튼이 hug로 줄며 auto-layout이 다음 행 y를 밀어 좌표 기준 조회가 깨짐). 사이드바 Cat Manage Row는 원래 FRAME이라 이 cascade 없음. **색상 결정(테이블 vs 사이드바 배경 차이 반영)**: 테이블(흰 배경)은 브리프 지시대로 텍스트 색상 코딩 — "수정"=`color/text-accent`(teal/700, 바인딩), "삭제"=로컬 톤다운 코랄(계산값 `#D9335A`, 흰 배경 대비 약 4.59:1 PASS, 새 토큰 아님·인라인 값). 반면 **사이드바(teal 배경)는 이미 확정된 전역 규칙("사이드바 안의 모든 텍스트는 always text-primary(ink), 예외 없이") 때문에 브랜드색 텍스트를 얹으면 teal-on-teal은 사실상 안 보이고 coral도 저대비라 브리프의 색 매핑을 그대로 적용하지 않고, 기존 Row Action Button의 배경색 구분(수정=흰 박스/삭제=코랄 박스, 둘 다 1px 잉크 보더+radius/xs+8×4padding 유지)만 남기고 아이콘만 ink 텍스트로 교체 — 텍스트 자체(단어)가 동작을 명시하고 색은 배경 박스가 담당하도록 조정. 이 판단 근거를 스크린샷+대비 계산으로 확인, design-pl 브리프의 문면과 다르게 적용한 부분이라 여기 명시. 폭 44×44 고정도 지시대로 outer frame `primaryAxisSizingMode/counterAxisSizingMode='AUTO'`로 hug 전환(테이블 37×21 내외로 자연 축소, 사이드바도 동일). Table Row 높이가 62→39로 auto-layout cascade로 축소(action 버튼이 작아졌으므로 자연스러운 결과, 6행 전부 균일 확인, 헤어라인/정렬 회귀 없음).
- **결함2 — 검색 Input 아이콘 비율**: 실측 결과 `leading-icon` 인스턴스가 이미 24×24 FIXED/FIXED, `relativeTransform`이 `[[1,0,x],[0,1,y]]`(스케일 왜곡 없음), mainComponent(Icon/Search, `96:12`)와 크기 일치 — **이미 정상, 수정 불필요**(직전 라운드에서 이미 해소된 상태를 재확인). Empty는 `Show Search Icon=false`라 아이콘 자체가 비표시(브리프 범위상 문제 없음).
- **결함3 — "새 카테고리 추가" 버튼 색 빠짐**: `docs/design/confirmed/b-2-contacts-layout.md` 6절 원문 재확인 결과 design-prompter 판단대로 Primary teal이 정답 맞음(amber 아님). 실측 결과 fill은 이미 정확히 teal(컴포넌트 `component/button-bg-primary` 바인딩, variant도 Style=Primary 정상) — 브리프가 가정한 (a)오버라이드 삭제/(b)variant 오선택 둘 다 아니고, **버튼 fill이 사이드바 배경과 완전히 동일한 teal이라 경계가 안 보이는 것이 실제 원인**(Button 컴포넌트 자체는 Primary variant에 보더가 없음 — 다른 화면 흰 배경에서는 문제 없는 정상 설계). 컴포넌트/색상 변경 없이 **인스턴스 레벨 오버라이드로 1.5px 잉크 보더 추가**(confirmed 2절 "1.5px 개별 버튼" 두께 재사용, `color/border-ink` 바인딩) — With Data(`186:358`)/Empty(`186:695`) 둘 다 적용, 스크린샷으로 사이드바 위에서 버튼 윤곽이 뚜렷이 분리됨을 확인. 본문 "연락처 추가"(amber) 버튼은 미변경.
- **결함4 — Select 드롭다운 화살표 이질감**: 실측 결과 chevron은 벡터가 아니라 **TEXT 레이어**(`▾` 유니코드 문자, Noto Sans KR Regular 12px, gray/600 색). 새 벡터 생성 없이(그리기는 graphic-designer 소관) 텍스트 속성만 오버라이드: `fontName.style`→Bold(두께감 대체 수단, 실제 strokeWeight 속성은 텍스트에 존재하지 않아 굵기로 근사), `fills`→`color/border-ink` 바인딩. With Data(`177:511`)/Empty(`177:716`) 둘 다 적용, 스크린샷으로 진하고 어두운 톤 확인.
- **회귀 확인**: 3화면 전체 스크린샷 재확인 — Login 완전 무수정, With Data/Empty 사이드바 teal 풀블리드·워드마크·테이블 앰버 헤더·헤어라인·민트 배너·활성 카테고리 amber 전부 정상 유지. Cat Actions/Table Row action 두 라벨 겹침 없음(메타데이터로 x좌표 간격 확인).
- 하지 않은 것: 마스터 컴포넌트 4종 미수정, design-systems/design-qa 미호출, 결함2는 이미 정상이라 판단해 변경 없음(임의로 재작업하지 않음), 새 토큰/아이콘 생성 없음(코랄 톤다운은 로컬 인라인 값, 텍스트 전용).

### 2026-07-13 — SCREENS 결함 수정 라운드 2 (Row Action Button/Search Icon/Header Logout 화면 반영)
- 배경: design-systems가 컴포넌트 레벨 3건(Row Action Button `166:421` Danger variant Icon/Edit→Icon/Delete 오배선 정정, Input `100:46`에 `Show Search Icon`+`Icon` 프로퍼티 신설, Button `97:47`에서 헤더 로그아웃=Style=Primary+Content=IconText+Icon=Icon/Logout 사용 패턴 확정)을 완료한 직후 호출됨. 대상은 파일럿 3개 중 Login(`153:19`, 무수정)/Contacts — With Data(`153:373`)/Contacts — Empty(`153:547`).
- **1. Row Action Button 인스턴스 교체가 아니라 "숨은 인스턴스 레벨 오버라이드" 버그였음**: `get_metadata`로 사전에 이미 전 행이 Row Action Button 인스턴스로 교체돼 있음을 확인(브리프의 "텍스트로 처리된 자리" 추정과 달리 실제로는 이미 인스턴스). 그러나 `componentProperties`를 직접 조회해 **모든 edit-action(Neutral) 인스턴스가 Icon 프로퍼티를 `96:27`(Icon/Delete)로 강제 오버라이드**하고 있어 컴포넌트 레벨 수정(Danger만 Delete로 정정)과 무관하게 편집 버튼에도 삭제 아이콘이 뜨는 실제 결함을 발견 — 원인은 인스턴스 오버라이드지 컴포넌트 버그가 아님. `setProperties({'Icon#166:0':'96:22'})`로 edit-action 전체(Table Row 6행 × With Data + 사이드바 Cat Actions 4행 × 2화면 = 14곳)를 Icon/Edit로 명시 재설정. delete-action(Danger)은 이미 `96:27`로 정상 렌더링 중이라 미변경. `findOne(name==='icon')` 깊은 조회로 edit=Icon/Edit / delete=Icon/Delete 전수 재확인 완료(스크린샷도 펜/휴지통 아이콘이 시각적으로 다름을 확인).
- **2. 검색 Input `Show Search Icon=true`**: With Data 검색 인풋(`177:499`)만 토글(Empty는 브리프 범위 밖이라 미적용, 스크린샷으로 Empty 쪽엔 아이콘 없음 확인). 높이 31→38px 자동 확장 확인.
