# Brand Guide — 확정 브랜드 정체성

이 문서는 이 프로젝트의 **현재 확정된** 브랜드 정체성을 담는다. 다른 모든 에이전트가 브랜드 값을 참조하는 소스 오브 트루스이며, brand-designer가 브랜드 방향을 정식화·변경할 때마다 이 파일을 **덮어써서** 항상 최신 상태로 유지한다(로그가 아니다 — 지난 결정은 git 히스토리에 남는다).

## 1. 브랜드 트라이앵글

- **포지셔닝**: 차가운 CRM형 유틸리티가 아니라, 사용자가 직접 손으로 정리한 "내 사람들 목록" 같은 정서적 온기 + 로그인 사용자별 데이터 격리(신뢰 요소)를 시각적 경계로 드러낸다.
- **퍼스낼리티**: "다정하지만 경계는 또렷한 친구" — 이번 확정 디자인에서는 이 퍼스낼리티가 구체적으로 **네오브루탈리즘(두꺼운 잉크 아웃라인 + 하드 오프셋 그림자) + 8비트 픽셀 컨페티(장식용 미니 픽셀 아이콘 산포)**라는 두 트렌드의 하이브리드로 표현된다. 즉 "다정함"은 파스텔이 아닌 채도 있는 원색(스카이블루/코랄/앰버)과 장난스러운 픽셀 장식으로, "경계는 또렷하게"는 2~3px 굵은 잉크 보더와 그림자의 딱 떨어지는 오프셋으로 표현된다.

### 2026-07-17 각주 — Primary/Secondary/Accent 역할 재평가 (실측 근거)

8개 확정 프레임(main/main-수정/main-삭제/main-검색없음/Join/login/login-알림창/main-알림창, 신규 부모 섹션 `501:2505`) 전체를 `use_figma` 읽기 전용 스크립트로 순회해 스카이(`#1395E6`)/틸(`#17A398`)/코랄(`#FF5A76`)/앰버(신값 `#FFCE2C` + 구값 `#FFCB47`) 4색의 fill·stroke를 노드 단위로 수집하고 면적(px²)·용도를 실측했다 — design-pl의 초기 스팟체크("sky 16곳, teal 8곳" 관찰)를 8프레임 전체·fill+stroke 양쪽으로 검증하는 라운드였다.

- **스카이블루 — Primary(신규 편입)**: 사이드바 전체 배경(main/main-수정/main-삭제/main-검색없음/main-알림창 5개 프레임 × 121,472px², 각 프레임의 8.13%) + Join/login/login-알림창 풀블리드 배경(3개 프레임 × 프레임 전체 100%) + 아바타 원 배경(인스턴스 로컬 오버라이드) + 인증 카드 하단 액센트 스트립(옛 틸 자리를 대체). 실측 면적이 나머지 3색을 압도적으로 초과하는 유일한 "구조적 지배색"이며, 사이드바·풀블리드 배경·아바타처럼 화면 전반을 지배하거나 브랜드 식별의 핵심 요소에 쓰인다는 "지배적 역할" 기준에 정확히 부합한다.
- **코랄 — Secondary(유지)**: 검색 버튼, row "삭제" 액션 아웃라인, "삭제하기" CTA(main-삭제), 삭제 경고 박스(코랄 8% 틴트 배경+보더), 로고 심볼 채움(모든 화면 공통), 가족 카테고리 배지 보더/닷. 8개 프레임 전반에 걸쳐 "파괴적 액션 + 브랜드 마크"라는 일관된 역할로 반복 등장하며, 스카이 다음으로 큰 실측 면적을 차지한다.
- **앰버 — Accent(유지, 값 드리프트 발견)**: 모든 주요 CTA(가입하기/로그인/저장하기), 카드·모달 상단 액센트 스트립, 활성 사이드바 nav에 일관되게 쓰인다. 절대 면적은 크지만(대형 CTA 버튼 크기 때문) 배경/구조색으로는 한 번도 쓰이지 않고 항상 "지금 눌러야 할 그 액션 하나"를 강조하는 좁은 용도에 국한돼 Accent 역할이 정확히 맞다. **부수 발견(재바인딩은 이번 라운드 범위 밖, 관찰만 기록)**: count pill(비활성 nav)과 main-삭제 모달 상단 스트립은 신값(`#FFCE2C`)이 아니라 구값(`#FFCB47`)에 머물러 있다 — Stage2 앰버 값 전환이 이 두 곳에는 아직 반영되지 않은 값 불일치. design-systems가 참고할 관찰로 남긴다.
- **틸 — Primary에서 제외, "카테고리 식별색(narrow)"으로 재정의**: 8개 프레임 전체에서 틸의 유일한 **가시적** 용도는 (1) CatBadge "회사" 카테고리 보더/닷(1px 스트로크), (2) row 액션 버튼 "수정" 아웃라인(1px 스트로크, 6개 행마다 반복되지만 개별 면적은 미미)뿐이다. App 배경 레이어와 Avatar 래퍼 Container의 틸 fill 자체는 데이터상 여전히 존재하지만, 둘 다 스카이블루가 그 위에 덧씌워진 비가시 레이어로 남아있어 실제로는 렌더링되지 않는다(`docs/design/graphic-assets.md`의 "Avatar가 신 세대에도 여전히 틸" 기록은 **아이콘 글리프 색**에 대해서만 맞고, **원 배경 색**에는 틀렸다는 점이 이번 재검증으로 재확인됐다 — 원 배경은 스카이블루 로컬 오버라이드다). Secondary/Accent 후보로 격상하지 않는다 — 이미 코랄·앰버가 각각 그 역할을 맡고 있고, 틸의 용도가 "회사" 카테고리와 "수정" 액션 두 곳으로 좁게 고정돼 있어 트라이어드에 편입할 실측 근거가 없다. 이 판단은 design-pl의 초기 관찰("teal이 작은 용도에 국한")과 일치하며, 이번 라운드는 그 관찰을 8프레임 전체 실측으로 확정한 것이다.
- **프리미티브 토큰 이름은 변경하지 않는다** — `color/teal/500`, `color/sky/500`, `color/coral/500`, `color/bg-cta-amber` 등 기존 토큰 이름 그대로 유지한다. 바뀌는 건 "이 색이 브랜드 트라이어드에서 어떤 역할을 맡는가"라는 문서상의 역할 매핑뿐이며, Figma 컴포넌트·토큰의 실제 바인딩은 이번 라운드에서 전혀 손대지 않았다(이미 과거 라운드에서 완료된 리바인딩 결과를 역할 언어로 재문서화만 함).

**위계 정리**: Display(워드마크, Baloo 2) → Heading(Noto Sans KR Black) → 마이크로 라벨(Inter Black, 대문자+강한 자간 — 한글 라벨에도 동일 적용됨, 재현 시 그대로 따를 것) → Body/Button(Noto Sans KR Bold/Regular) → Caption(Noto Sans KR Regular, 뮤트 컬러)의 4~5단계 위계. 세 개 폰트 패밀리(Baloo 2 / Inter / Noto Sans KR)를 역할별로 엄격히 분리해서 쓰는 것이 이 컨셉의 특징이다 — 아무 텍스트에나 섞어 쓰지 않는다.

## 4. 보더 & Radius 스케일

**보더 두께**:
- **3px** — 구조적 대분할선: 사이드바 우측 보더, 메인 헤더 하단 보더
- **2px** — 기본 컴포넌트 보더: 버튼, 입력창, 모달/카드 외곽, count pill, 로고 원, 아바타 원, CornerInput 코너 브래킷
- **1px** — 경량 보더: CatBadge, row 액션 아웃라인 버튼(수정/삭제), 카테고리 관리 소형 버튼 보더(`#1c1f21`), 요약 박스 보더(`#ede6d8`), 경고 박스 보더(코랄), 테이블 헤더 하단(2px, ink) — 단 테이블 행 사이 구분선만 예외로 연한 베이지 `#ede6d8` 1px 헤어라인

**Radius 스케일**:
- **999px(full round)** — 로고 원, 아바타 원, count pill, CatBadge 닷, TypeSelector 닷, 보조 버튼 아이콘 배지(20px)
- **10px** — 사이드바 nav 버튼, NeoInput/NeoBtn/NeoSelect(검색·필터·추가), 모달 저장/취소 버튼, 인증 폼 주요 버튼
- **8px** — 헤더 로그아웃 버튼, CatBadge, "총 N건" pill, row 액션 아웃라인 버튼(수정/삭제), TypeSelector 칩
- **5px** — 카테고리 관리 소형 액션 버튼(연필/휴지통)
- **6px** — 삭제 모달 경고 박스
- **0px(각짐)** — 인증 폼 CornerInput의 입력창 자체(코너 브래킷 장식과 짝을 이루려 의도적으로 각짐 처리, 목록 화면 NeoInput은 반대로 10px 라운드 — 컨텍스트별로 다른 처리), 삭제 모달 요약 박스, 알림 배너(성공/에러 둘 다 각진 사각형, 카드 시스템과 다르게 라운드 없음)

## 5. 그림자 시스템 — 이전 문서의 "그림자 없음" 기록은 폐기

이 확정 디자인은 **두 종류의 그림자를 명확히 구분해서 쓴다**:

1. **하드 오프셋 "스티커" 그림자**(neo-brutalist) — 블러 없이 순수 오프셋, 색상은 항상 ink `#1a1a1a`. `drop-shadow(Npx Npx 0px #1a1a1a)`.
   - 1px: count pill(비활성)
   - 2px: 주요 CTA 버튼(검색/전체/추가/로그인/가입하기/저장하기/삭제하기), 활성 사이드바 nav, 모달 닫기(X) 버튼
   - 6px: 모달 카드, 인증 카드(Join/login) 전체
   - **적용 안 됨**: 보조/중립 버튼(취소, 로그아웃 NeoBtn, "로그인으로 돌아가기"), 비활성 nav, CatBadge, row 아웃라인 액션 버튼 — 주요 액션과 보조 액션을 그림자 유무로 구분하는 것이 일관된 규칙이다.
2. **소프트 블러 그림자**(토큰 기반) — 알림 배너(성공 토스트 `247:5558`, 에러 배너 `247:5303`) 전용. `0px 2px 8px rgba(28,31,33,0.08)`. 하드 스티커 그림자와 달리 오프셋 없이 부드럽게 퍼져, "일시적으로 떠 있다 사라지는" 알림의 성격을 카드/버튼과 다르게 표현한다.

## 6. 간격 리듬

4px 그리드 기반. 사이드바 내부 블록 간 44px(로고~카테고리), 카테고리 항목 간 4px gap(pt 8px), 카테고리 관리 항목 간 6px gap. 메인 콘텐츠: 헤더 padding 12/16/20px, 검색행~추가행~리스트 요약행 간 각 68/144/180px 오프셋(대략 16~24px 시각적 gap). 인증 카드: 필드 간 20px, 필드 라벨~입력창 6px, 카드 padding 32px(좌우)/28~32px(상하). 모달: 필드 간 20px, 카드 padding 32px(좌우)/24~28px(상하), 버튼 행 pt 4px + gap 8px. CornerInput 44px 높이, padding 14~16px.

## 7. 장식 모티프 (Decorative Motifs)

- **픽셀 컨페티 배경(BgPixels)** — Join/login처럼 브랜드색(스카이블루) 풀블리드 배경에서는 `PixelDiamond`/`PixelStar`/`PixelCross` 8bit 픽셀 아이콘 약 14개를 화면 전역에 8~20px 크기, 25~40% 불투명도로 무작위 산포시킨다. 여기에 점선 원(Ellipse, 대시 아웃라인) 3개를 카드 주변에 추가로 배치한다. 카드 안쪽 폼 하단에도 8px 크기 Diamond/Star 5개를 25% 불투명도로 한 줄 배치("컨페티 푸터").
- **픽셀 클러스터(MemphisAccents)** — 사이드바처럼 좁은 브랜드색 패널에서는 같은 픽셀 요소(Diamond/Star/Vector/Ellipse)를 화면 전역이 아니라 패널 하단 한 구석에 작게 클러스터로 모아 10% 컨테이너 불투명도(+개별 요소 35~67%)로 은은하게 배치한다. **풀블리드 배경 = 전역 산포, 좁은 패널 = 구석 클러스터**로 크기에 따라 밀도 규칙이 다르다.
- **코너 브래킷 인풋(CornerInput)** — 인증 폼과 편집 모달의 입력창은 각진(radius 0) 흰 박스 바깥 네 모서리에 8×8px, 2px ink 보더의 "ㄱ자" 브래킷을 얹어 "타겟팅/청사진" 느낌을 낸다. 이 장치는 인증/모달의 필드에만 쓰고, 목록 화면의 검색·필터 NeoInput(10px 라운드, 브래킷 없음)에는 쓰지 않는다.
- **카드 상/하단 액센트 스트립** — 모달과 인증 카드는 흰 카드 상단에 8px 두께 앰버(`#ffcb47`) 스트립을 얹는다(border-b 2px ink). 인증 카드(Join/login)는 여기에 더해 카드 **하단**에도 8px 두께 스카이블루(`#1395e6`, ⚠2026-07-16 재실측 정정 — 원문은 "틸"이었으나 실제로는 스카이블루) 스트립을 하나 더 얹어 위/아래를 다른 브랜드색으로 "샌드위치"한다 — 모달에는 이 하단 스트립이 없다(상단 앰버만, 단 main-삭제 모달의 상단 스트립은 구값 `#FFCB47`에 머물러 있음 — 2절 각주 참고).
- **Count Pill / CatBadge** — 원형 알약형 카운트 배지(사이드바 nav, "총 N건")와 연한 배경+보더+닷의 카테고리 배지(CatBadge, 2번 참고)가 리스트 전반에서 반복되는 핵심 장식 겸 정보 전달 장치다.
- **로고 심볼 내부 픽셀 스타** — 코랄 원 로고 심볼은 빈 원이 아니라 내부에 흰색 12px `PxStar` 픽셀 별 아이콘을 담고 있다(아래 8번 참고).

## 8. 로고/워드마크 — 정확 재현 스펙

**심볼**: 지름 28px 원, 채움 `#ff5a76`(코랄), 2px ink(`#1a1a1a`) 스트로크, 내부에 흰색 `PxStar`(픽셀 별) 아이콘 12px 중앙 배치(패딩 2px). **빈 원이 아니다 — 항상 픽셀 별을 품는다.**

**워드마크**: "yourbook." — 전부 소문자 + 마침표(대문자 없음). 서체 **Baloo 2 ExtraBold**(Bold 아님 — 이전 문서의 "Bold, ExtraBold 아님" 기록은 이번 관찰로 정정됨), 크기 22px, 자간 -0.55px(약 -2.5%, 살짝 좁힘). **색상은 배경에 따라 반전된다** — 스카이블루 배경(사이드바, 인증 카드 밖) 위에서는 흰색, 흰 배경(카드 내부, 헤더) 위에서는 ink `#1a1a1a`(⚠ 2026-07-17 정정 — 원문은 "틸 배경"이었으나 재확정 세트에서 해당 배경이 스카이블루로 전환됨, 반전 규칙 자체는 동일하게 유지). 고정된 딥틸 색(`#0F7A6E`) 하나만 쓴다는 이전 기록은 폐기한다.

**배치**: 심볼과 워드마크는 가로 배치, gap 10px(이전 문서의 14px 기록에서 정정), 아이템 정렬은 `items-center`로 심볼·텍스트 수직 중심 정렬.

**축소 시**: 심볼만 단독으로 쓸 경우(파비콘 등) 최소 크기에서도 2px 스트로크 비율은 유지하되, 내부 픽셀 별은 남긴다(관찰된 프레임에서는 항상 심볼+별 세트로만 등장, 스트로크를 제거한 별도 축소형은 이번 소스에 없음 — 필요 시 design-systems가 파비콘 전용 변형을 정의할 것).

## 9. 아이콘 · 일러스트레이션 방향

관찰된 아이콘은 명확히 **두 계층**으로 나뉜다. 이 구분을 유지해서 확장한다(graphic-designer 참고):

1. **마이크로 픽셀 아이콘(8~15px, `Pixel*`/`Px*` 네이밍)** — 8비트/픽셀아트 렌더링, 단색(주로 ink 또는 흰색, 배경에 따라 반전), 안티앨리어싱 없는 각진 실루엣. 두 가지 용도로 쓰인다:
   - **기능 마이크로 아이콘**: `PxSearch`(검색), `PxPlus`(추가), `PxLogout`(로그아웃), `PixelPencil`(수정), `PixelTrash`(삭제), `PixelEye`(비밀번호 표시/숨김) — 버튼/입력창 안에 인라인으로 들어가는 9~15px 크기.
   - **순수 장식 컨페티**: `PixelDiamond`, `PixelStar`, `PixelCross`, `Ellipse`(점선 원) — 7번의 장식 모티프 섹션 참고.
2. **표준 플랫 아이콘(24px, `Icon/*` 네이밍, 컴포넌트화됨)** — 픽셀아트가 아닌 매끈한 플랫 실루엣. `Icon/User`(아바타 내부), `Icon/Alert`(성공/에러 배너 공용 — 배너 종류는 아이콘이 아니라 배경색으로만 구분한다). **⚠ 2026-07-17 갱신 — Basic/Visual 트랙 재판정 완료**(`docs/design/graphic-assets.md` 최신 절 참고): Search/Add/Edit/Delete/Logout/User 6종은 면색 없이 2px 스트로크만 쓰는 **Basic** 트랙으로, Category/Alert 2종은 굵은 아웃라인+브랜드색 평면 채색을 유지하는 **Visual** 트랙으로 재분류됐다. 이 계층은 이미 그래픽 컴포넌트로 등록되어 있다(Icons 페이지 `96:7` 참조).

**빈 상태(empty state) 그래픽**: `main-검색없음`(`242:4280`, 구 ID — 신규 확정 세트 `501:4218`)에서 확인 — 별도의 큰 일러스트 대신 위 1번 계층의 픽셀 아이콘을 확대한 형태(깨진/기울어진 돋보기 실루엣)를 재사용한다. **⚠ 2026-07-16 재실측 정정** — 몸통 색은 원문 "틸"이 아니라 스카이블루(`#1395e6`)이고 크랙만 코랄 유지("스카이블루+코랄 2색"). 신규 마스코트를 만들지 않고 로고/기능 아이콘과 같은 픽셀아트 언어를 확장하는 방향이 이 컨셉과 맞는다.

**graphic-designer 방향 요약**: 신규 아이콘/오브제를 그릴 때 (a) 9~15px 기능 아이콘은 픽셀아트 계열로, (b) 24px 이상 아바타·상태 아이콘은 Basic(면색 없는 스트로크)/Visual(브랜드색 평면 채색) 두 트랙 중 대상의 반복 빈도·브랜드 표현 필요성에 따라 선택해서, (c) 장식용 소품은 Diamond/Star/Cross/dashed-circle 어휘 안에서 확장한다. 3D/아이소메트릭/스큐어모피즘은 이번 확정 디자인에 없으므로 배제한다.

## 10. 알림/토스트 오버레이 — 의도된 플로팅 UI (버그 아님, 사용자 확인 완료)

`login-알림창`(`247:5303`)의 에러 배너와 `main-알림창`(`247:5558`)의 성공 토스트는 **각각 login/main 화면 위에 최상위 z-index로 플로팅하는 의도된 오버레이**다 — 레이아웃 버그가 아니라 사용자가 직접 확정한 디자인 의도다.

- **에러 배너**(`247:5303`): login 인증 카드 내부, 상단 앰버 스트립 바로 아래에 카드 폭(416px)에 딱 맞게 절대 위치로 얹힌다. 폼 콘텐츠의 레이아웃 흐름에 영향을 주지 않고 그 위에 겹쳐 뜬다. bg `#ffe4e8`, 소프트 블러 그림자, `Icon/Alert` 24px + 본문 텍스트 한 줄.
- **성공 토스트**(`247:5558`): 메인 화면의 사이드바를 제외한 콘텐츠 영역 상단에, 검색행 자리를 그대로 덮으며 절대 위치로 뜬다(검색행 레이아웃은 안 밀려나고 토스트가 그 위에 겹친다). 폭은 메인 콘텐츠 전체(812px), bg `#d8fff5`, 소프트 블러 그림자, `Icon/Alert` 24px + "추가되었습니다"(1차, Medium 13px) + "방금 · 잠시 후 사라짐"(2차, Regular 11px, 자동 소멸 안내).

**후속 작업자(ui-designer/interaction-designer) 참고**: 두 배너 모두 하단 콘텐츠를 밀어내지 않는 오버레이 방식으로 구현한다. 자동 소멸 타이밍/애니메이션은 interaction-designer·motion-designer의 몫이며 이 문서는 정적 배치·스타일만 규정한다.

## 11. 텍스트 대비 규칙

브랜드 3색(스카이블루/코랄/앰버)은 전부 mid-tone이라 흰 텍스트와 페어링하면 대비가 약할 수 있다 — 실제 확정 디자인에서도 앰버/코랄 배경 위 텍스트는 항상 ink(`#1a1a1a`)를 쓰고, 흰 텍스트는 스카이블루(브랜드색 중 가장 어두운 톤) 배경 위에서만 쓴다(사이드바, 로그아웃 버튼 등). 브랜드색을 배경으로 쓸 때 그 위 텍스트 색은 배경 밝기를 보고 ink/흰색 중 선택하는 규칙을 유지한다.

## 12. 워드마크 네이밍 미확정 — 중요

"yourbook."은 서체·톤 표현용 placeholder이며 최종 서비스명이 아니다. 실제 서비스명 확정 시 design-pl 경유 별도 논의가 필요하다 — brand-designer가 임의로 확정하지 않는다.

## 13. 이전 시안 이력

이전에 진행됐던 AI 파일럿 라운드(1차 "Warm Ledger" 등 3개 시안, 2차 "B-2" 레이아웃)는 "브랜드 컨셉 Concepts" 페이지(`34:2`)에 그대로 보존되어 있다 — 삭제하지 않았다. 다만 이 문서(Brand Guide)의 기준 소스는 더 이상 그 파일럿이 아니라 이번 사용자 직접 확정 디자인(8개 프레임)이다. 파일럿과 이번 확정 디자인 사이의 구체적 차이(그림자 유무, 워드마크 웨이트/색상, 심볼-워드마크 간격 등)는 위 각 섹션에 "이전 기록 정정"으로 명시했다.

> **⚠ 2026-07-17 각주 (design-qa 정정) — Color Palette / Triad Swatch Row 높이 통일.** design-qa 감사에서 MEDIUM 결함 1건 발견: `921:5` "Triad Swatch Row" 안 4개 스와치 컨테이너(Primary `921:6` / Secondary `921:10` / Accent `921:14` / Ink `921:18`)의 폭은 196px로 동일했지만, 설명 텍스트 줄 수 차이(3줄 vs 2줄)로 hug 사이징 결과 높이가 166px/153px로 13px 어긋나 있었다 — `docs/harness/design-team/figma-page-format-guide.md` 1번("같은 카탈로그 안의 모든 스와치는 동일한 크기(width/height)를 쓴다") 위반. 정정 방식으로 텍스트 재작성(줄 수 강제 맞춤) 대신 **4개 컨테이너 모두 `primaryAxisSizingMode='FIXED'` + 높이 166px(기존 최댓값) 고정**을 채택했다 — 이 페이지의 스와치는 원래 "내용에 따라 흔들리지 않는 고정 크기"가 기준이라는 포맷 가이드 취지에 더 맞고, 설명 문구를 인위적으로 줄여/늘려 내용을 왜곡할 필요가 없기 때문이다. 색상 값·바인딩·텍스트 내용은 전혀 변경하지 않았다. 정정 후 4개 스와치 하단이 모두 정렬되고 잘림/겹침이 없음을 스크린샷으로 확인했으며, 상위 Row(`921:5`, 172px)·Color Palette 섹션(`921:2`, 444px) 높이는 원래도 최댓값(166px) 기준으로 계산돼 있어 변경되지 않았다.
