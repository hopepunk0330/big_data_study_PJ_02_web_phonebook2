# 누락 화면 목록 (Missing Screens)

이 문서는 확정 8개 프레임(Figma, "확정 디자인 - 절대 원본 건들지 말것")과 `docs/planning` 기획 문서를 대조한 커버리지 감사(2026-07-13, ux-designer)에서 발견한 누락 화면·상태와, 그 이후 사용자 결정(2026-07-14)으로 추가된 항목을 합쳐 정리한 목록이다.

**용도**: 3-B "전체 확장" 단계에서 design-pl / ui-designer가 어떤 화면을 새로 설계해야 하는지 참고하는 목록. **지금 이 문서 자체는 화면을 설계하거나 만들지 않는다** — 목록 정리만 한다. **전체 확장(9번) 착수는 사용자가 "디자인 시스템 최종 승인"을 별도로 확정해야 시작한다(`docs/harness/design-team/figma-file-organization.md` 3-B번 4-1항) — 이 문서의 항목이 갖춰졌다고 자동으로 착수하지 않는다. 2026-07-14 기준 아직 승인 전이라 대기 중이다.**

**상태 값**: 모든 항목은 현재 "설계 대기" — 아직 Figma에 프레임이 없고, 구체적 레이아웃도 확정되지 않았다.

**문서 버전 안내 (2026-07-14 갱신)**: 화면정의서 소스는 `docs/planning/02_연락처관리_웹서비스_화면정의서_v1.1.md`다(구 `_v1.0.pdf`/임시 `_v1.0.md`는 폐기, PDF 재변환 없이 md 버전만 올려서 참조하는 방식으로 확정됨). 아래 항목들의 "근거"가 `_v1.0`으로 표기된 곳은 전부 `_v1.1.md`를 가리키는 것으로 갱신했다 — 앞으로 이 문서를 참조하는 모든 팀원은 `_v1.1.md`(또는 그 이후 최신 버전)를 봐야 한다.

## 8. 비밀번호 찾기 플로우 (신규, 사용자 결정 — 기능 범위 확장)

- **근거**: 로그인 화면(login, 247:6666)의 "비밀번호 찾기" 링크는 확정 프레임에 이미 존재. 그러나 원래 `docs/planning/04_연락처관리_웹서비스_PRD_v1.0`에서 이 기능은 비목표(N1)였음. 사용자가 2026-07-14에 이를 기능 범위에 포함하기로 새로 결정함(단순 라벨링이 아니라 실제 기능 확장).
- **비고**: 이메일/전화번호 입력 → 인증 → 재설정으로 이어지는 구체적 단계는 아직 미정. **PRD 문서(`docs/planning`) 자체의 N1 항목 수정은 디자인팀 범위 밖 — 메인 세션이 별도로 처리한다.** 이 목록에는 디자인팀이 참고할 "누락 화면"으로만 추가.
- **상태**: 설계 대기
