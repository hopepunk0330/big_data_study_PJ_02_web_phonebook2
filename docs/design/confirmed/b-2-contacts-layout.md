# 확정 스펙 — B-2 (연락처 관리 화면 레이아웃)

이 문서는 `docs/design/figma-file-organization.md` 2-3번 규칙에 따라 작성된 **확정 스펙 문서**다. `docs/design/brand-guide.md`가 컬러 hex·폰트명 수준의 헤드라인 요약이라면, 이 문서는 그보다 더 구체적인 근거다 — **이 문서만 읽어도 B-2를 그대로 재현할 수 있어야 한다.** 두 문서의 내용이 다르면 이 문서를 따른다.

## 확정 배경

- **원본 위치**: Figma "브랜드 컨셉 Concepts" 페이지(fileKey `zgGlMBwFglaDlaeyP4CkgR`, pageId `34:2`), 2차 레이아웃 시안 프레임 `62:6` ("B-2").
- **확정 경위**: 1차로 Concept B "Warm Ledger"(톤 프레임 `34:4`, 52:2/52:3에 정식화)가 확정된 뒤, 2차 레이아웃 정밀화 라운드에서 3개 시안(B-1/B-2/B-3) 중 **B-2**(좌측 사이드바 내비 + 테이블형 리스트, 고밀도 정보 배치)가 최종 확정됐다.
- **관찰 근거**: brand-designer가 `34:4`와 `62:6`을 `get_screenshot`/`get_design_context`로 직접 다시 열람해 관찰한 내용을 Brand Guide 페이지("레이아웃 관례 — B-2" 섹션, 노드 `125:2`)와 `docs/design/brand-guide.md`에 정식화했다. 이 문서는 그 관찰 내용을 ui-designer가 실행 가능한 형태로 구체화·구조화한 것이다.
- **문제 진단(왜 이 문서가 필요한가)**: 이전에 ui-designer가 SCREENS 파일럿을 만들 때 B-2를 "그대로 이어받아 발전"시키지 않고 컬러/느낌만 참고해 재해석해버린 사례가 있었다. 원인은 B-2의 레이아웃 관례(사이드바 구성, 테이블 행 스타일, spacing·radius 등)가 문서화되어 있지 않아 재현할 근거가 없었기 때문이다. 이 문서는 그 공백을 메운다.

## 적용 범위 — 어떤 화면에 어떻게 적용하는가

이 문서가 규정하는 **사이드바 + 테이블형 리스트 구조**는 연락처 목록을 다루는 화면(SCREENS "Contacts — With Data", "Contacts — Empty")에 그대로 적용한다. 반면 **보더 위계·라운드 스케일·간격 리듬·타이포 스케일·색상 로직**은 B-2에서 확립된 공통 디자인 언어이므로, 구조가 다른 화면(SCREENS "Login" — 사이드바·테이블 없음)에도 동일하게 적용한다. 즉:

| 화면 | 사이드바+테이블 구조 | 보더/라운드/간격/타이포/색상 규칙 |
|---|---|---|
| Login | 적용 안 함(단일 카드형 폼) | 그대로 적용 |
| Contacts — With Data | 그대로 적용(아래 구조 섹션) | 그대로 적용 |
| Contacts — Empty | 그대로 적용, 단 테이블 바디 대신 빈 상태 그래픽(아래 "빈 상태" 절 참고) | 그대로 적용 |

## 1. 레이아웃 구조 (Structure)

### 1-1. 좌측 사이드바 내비게이션

- **폭**: 고정 약 212px, 화면 세로 전체를 풀블리드로 채운다. 배경은 주조색 teal(`#17A398`) 전면 채색 — 흰 배경이나 그라데이션 없음.
- **내부 블록 순서(위→아래)**, 블록 간 gap 16px:
  1. **브랜드 로고 영역** — 원형 심볼 배지 + 워드마크(24px 이상 크기이므로 잉크 아웃라인 동반, Brand Guide 로고 규칙 참고). 사이드바 상단에 배치.
  2. **"CATEGORY" 내비게이션 목록** — 항목마다 라벨 + 우측 정렬 pill형 카운트 배지(radius 999). 활성 항목은 흰 배지, 비활성 항목은 앰버(`#FFCB47`) 배지 — 배경(teal)과 항상 반전 대비를 이루도록 한다(흰 배지는 어두운 텍스트, 앰버 배지도 어두운 텍스트).
  3. **"카테고리 관리"** — 각 카테고리 항목에 인라인 수정/삭제 소형 액션.
  4. **새 카테고리 추가 폼** — 입력창 1개 + 추가 버튼, 사이드바 최하단.
- 사이드바 안의 모든 항목은 브랜드색(teal) 배경 위이므로 텍스트는 항상 `text-primary`(ink `#1C1F21`) — 흰 텍스트 금지(AA 위반, Brand Guide 대비 규칙).

### 1-2. 본문 (테이블형 리스트, 고밀도)

- **컨테이너 padding**: 20px (여백을 넉넉히 주는 미니멀 스타일과 반대 방향 — 고밀도 정보 배치 우선).
- **블록 순서(위→아래)**, 블록 간 gap 14px:
  1. **상단바** — 화면 타이틀(좌) + 로그인 사용자명 + 로그아웃 버튼(우).
  2. **검색행** — 검색 입력창 + 검색 실행 버튼(coral, 파괴적 액션과 함께 coral을 쓰는 유일한 비파괴 예외).
  3. **"추가" 행** — 여러 입력 필드(이름/전화번호/주소/카테고리 등)를 하나의 그룹 컨테이너 박스로 묶고, 그 안에 추가 버튼(amber) 배치. 이 박스는 2.5px 잉크 보더 그룹 컨테이너로 테이블과 동일한 위계.
  4. **리스트 헤더** — 총 건수 표시(예: "총 24건").
  5. **테이블** — 헤더 행 + 데이터 행. 헤더는 앰버 틴트(`#FFF0CC`) 배경.
  6. **상태 메시지 배너** — 성공/오류/안내 메시지, 옅은 민트 틴트(`#E5F0ED`) 배경.

### 1-3. 빈 상태 (Contacts — Empty)

- 사이드바 구조는 동일하게 유지한다(내비게이션은 데이터 유무와 무관하게 항상 노출).
- 본문 테이블 헤더까지는 그대로 두되, 데이터 행 대신 **빈 상태 그래픽**을 배치한다. Brand Guide 일러스트레이션 방향에 따라 마스코트를 새로 그리지 않고 **로고 원형 심볼을 확대 재사용**한다(graphic-designer에게 새 그래픽 요청 불필요).
- 검색행/추가행은 데이터 유무와 무관하게 항상 노출(카테고리가 비어 있어도 새로 추가할 수 있어야 하므로).

## 2. 보더 위계 (Border Weight) — 그림자 전혀 쓰지 않음

elevation(그림자)은 이 레이아웃에서 전혀 사용하지 않는다. 구조적 중요도는 오직 잉크 보더(`#1A1A1A`) **두께**로만 표현한다.

| 두께 | 적용 대상 |
|---|---|
| 3px | 화면 최외곽 프레임 |
| 2.5px | 그룹 컨테이너 (테이블 전체, "추가" 행의 다중 입력 필드 묶음 박스) |
| 1.5px | 개별 입력창·버튼 |
| 1px | 행 내부 소형 액션 버튼(수정/삭제) |
| 1px, 색상 `#E0E0E0` (잉크 아님, 예외) | 테이블 행 사이 구분선(헤어라인) — 고밀도 리스트의 시각적 소음을 줄이기 위한 유일한 예외 |

이 5개 값(3px/2.5px/1.5px/1px잉크/1px헤어라인) 밖의 임의 보더 두께를 추가하지 않는다.

## 3. 라운드 스케일 (Corner Radius)

| radius | 적용 대상 |
|---|---|
| 16px | 화면 최외곽 프레임 |
| 10px | 테이블/그룹 박스 |
| 8px | 버튼·입력창·내비게이션 아이템·상태 배너 |
| 5px | 행 내부 소형 액션 버튼 |
| 999px (pill) | 카운트 배지 |

이 5단계 밖의 임의 radius 값을 추가하지 않는다.

| 본문 블록 간 gap | 14px |
| 테이블 행/헤더 padding | 12×9px (가로×세로) |

| 일반 버튼 padding | 12×7px |

| 행 1차 식별자(이름) | 11.5px, Bold | |

## 6. 색상 사용 로직 (Color Logic in UI)

| 색상 | 용도 |
|---|---|
| Primary teal `#17A398` | 주요 액션(추가/로그아웃 버튼) + 데이터 강조 텍스트(카테고리 값 인라인) |
| Secondary coral `#FF5A76` | 파괴적 액션(삭제) + 검색 실행 버튼 |
| Accent amber `#FFCB47` | 선택된 내비게이션 상태 + 추가 버튼 + 테이블 헤더 틴트 배경 |
| 중립(흰/크림 `#F9F6F1`) | 배경 + 보조 액션(수정, 필터 리셋) |
| 민트 틴트 `#E5F0ED` | 상태 메시지 배너 전용 — 브랜드 3색 대신 옅은 톤으로 알림을 절제되게 표현 |

**텍스트 대비 절대 규칙**: 브랜드 3색(teal/coral/amber)은 전부 mid-tone이라 흰 텍스트와 페어링하면 AA 미달이다. 브랜드색 배경 위 텍스트는 항상 `text-primary`(ink `#1C1F21`) 고정 — 사이드바(teal 배경), amber 버튼, coral 버튼 모두 예외 없이 적용한다.

## 7. 장식 모티프 (Decorative Motif)

이 레이아웃 단계에서 반복되는 장식은 두 가지뿐이다:
1. **로고 원형 배지** — 사이드바 상단 1곳.
2. **pill형 카운트 배지**(radius 999) — 활성 행 흰 배지 / 비활성 행 앰버 배지, 배경과 항상 반전 대비.

기능 아이콘(검색/추가/수정/삭제/카테고리/로그아웃 등)은 graphic-designer가 이미 8종 제작 완료했다(`docs/design/graphic-assets.md` 참고, "Graphic Assets" 페이지 90:2, 노드 91:4~91:18). ui-designer는 이 원화를 임의로 새로 그리지 않고 그대로 가져다 조립한다.

## 8. ui-designer를 위한 안내

SCREENS 파일럿 3개(Login / Contacts — With Data / Contacts — Empty)를 제작할 때 이 문서를 **최우선 근거**로 삼는다. `docs/design/brand-guide.md`의 "레이아웃 관례 — B-2" 절보다 이 문서가 더 구체적인 실행 근거이며, 두 문서가 다르면 이 문서를 따른다. B-2를 "그대로 이어받아 발전(elevate)"시키는 것이 목표이며, 재해석·재설계가 아니다 — 위 수치·규칙에서 벗어난 임의의 spacing/radius/보더 값을 추가하지 않는다. design-systems 토큰은 아직 존재하지 않으므로(이 파일럿에서 사용된 값을 근거로 추후 추출됨), 이 문서와 Brand Guide의 원본 값을 직접 사용한다.
