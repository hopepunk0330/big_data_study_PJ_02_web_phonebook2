# 확정 스펙 — B-2 (연락처 관리 화면 레이아웃)

이 문서는 `docs/harness/design-team/figma-file-organization.md` 2-3번 규칙에 따라 작성된 **확정 스펙 문서**다. `docs/design/brand-guide.md`가 컬러 hex·폰트명 수준의 헤드라인 요약이라면, 이 문서는 그보다 더 구체적인 근거다 — **이 문서만 읽어도 B-2를 그대로 재현할 수 있어야 한다.** 두 문서의 내용이 다르면 이 문서를 따른다.

## 체크리스트 — 이 문서가 3대 영역을 모두 담고 있는지 자체 점검

이 문서는 ui-designer뿐 아니라 향후 graphic-designer/interaction-designer/ux-designer도 참고하는 공유 근거다. 아래 3가지가 전부 있어야 완성이다 — 참고하러 온 워커는 이 체크리스트로 "내가 필요한 정보가 다 있는지"를 먼저 확인한다.

- ☑ **로고 처리 스펙** — 9-2절 "정확한 워드마크 레터링 스펙"에 서체·크기·자간·색상·심볼 사양·간격·배치가 표로 정리되어 있음
- ☑ **퍼스낼리티 표현 장치** — 9절 전체(9-1 배지/스티커 형태 은유, 9-3 이중성 대비 장치)에 색/형태/보더 대비로 "다정함 vs 경계"를 표현하는 방법이 정리되어 있음
- ☑ **구조 규칙** — 1~6절(레이아웃 구조, 보더 위계, 라운드 스케일, 간격 리듬, 타이포, 색상 로직)에 spacing·radius·타이포 위계가 표로 정리되어 있음

## 확정 배경

- **원본 위치**: Figma "브랜드 컨셉 Concepts" 페이지(fileKey `zgGlMBwFglaDlaeyP4CkgR`, pageId `34:2`), 2차 레이아웃 시안 프레임 `62:6` ("B-2").
- **확정 경위**: 1차로 Concept B "Warm Ledger"(톤 프레임 `34:4`, 52:2/52:3에 정식화)가 확정된 뒤, 2차 레이아웃 정밀화 라운드에서 3개 시안(B-1/B-2/B-3) 중 **B-2**(좌측 사이드바 내비 + 테이블형 리스트, 고밀도 정보 배치)가 최종 확정됐다.
- **관찰 근거**: brand-designer가 `34:4`와 `62:6`을 `get_screenshot`/`get_design_context`로 직접 다시 열람해 관찰한 내용을 Brand Guide 페이지("레이아웃 관례 — B-2" 섹션, 노드 `125:2`)와 `docs/design/brand-guide.md`에 정식화했다. 이 문서는 그 관찰 내용을 ui-designer가 실행 가능한 형태로 구체화·구조화한 것이다.
- **문제 진단(왜 이 문서가 필요한가)**: 이전에 ui-designer가 SCREENS 파일럿을 만들 때 B-2를 "그대로 이어받아 발전"시키지 않고 컬러/느낌만 참고해 재해석해버린 사례가 있었다. 원인은 B-2의 레이아웃 관례(사이드바 구성, 테이블 행 스타일, spacing·radius 등)가 문서화되어 있지 않아 재현할 근거가 없었기 때문이다. 이 문서는 그 공백을 메운다.
- **출처가 두 개로 나뉘어 있다는 점에 유의**: 아래 1~6절(구조/보더/라운드/간격/타이포/색상)은 **62:6(B-2 레이아웃 시안)**을 관찰해 기록한 것이고, 9절(브랜드 퍼스낼리티 표현 장치)은 **34:4(1차 브랜드 톤 원본)**를 별도로 다시 관찰해 기록한 것이다. 두 관찰은 대상 프레임이 다르므로 규칙이 충돌하는 것처럼 보이는 지점(예: 보더 두께 규칙)이 있으면 9절 안의 "주의 — 출처 구분" 박스를 먼저 확인한다.

## 적용 범위 — 어떤 화면에 어떻게 적용하는가

이 문서가 규정하는 **사이드바 + 테이블형 리스트 구조**는 연락처 목록을 다루는 화면(SCREENS "Contacts — With Data", "Contacts — Empty")에 그대로 적용한다. 반면 **보더 위계·라운드 스케일·간격 리듬·타이포 스케일·색상 로직**은 B-2에서 확립된 공통 디자인 언어이므로, 구조가 다른 화면(SCREENS "Login" — 사이드바·테이블 없음)에도 동일하게 적용한다. 즉:

| 화면 | 사이드바+테이블 구조 | 보더/라운드/간격/타이포/색상 규칙 |
|---|---|---|
| Login | 적용 안 함(단일 카드형 폼) | 그대로 적용 |
| Contacts — With Data | 그대로 적용(아래 구조 섹션) | 그대로 적용 |
| Contacts — Empty | 그대로 적용, 단 테이블 바디 대신 빈 상태 그래픽(아래 "빈 상태" 절 참고) | 그대로 적용 |

## 1. 레이아웃 구조 (Structure)

### 1-1. 좌측 사이드바 내비게이션

- **폭**: 고정 약 212px, 화면 세로 전체를 풀블리드로 채운다. 배경은 주조색 teal(`#17A398`) 전면 채색 — 흰 배경이나 그라데이션 없음.
- **내부 블록 순서(위→아래)**, 블록 간 gap 16px:
  1. **브랜드 로고 영역** — 원형 심볼 배지 + 워드마크(24px 이상 크기이므로 잉크 아웃라인 동반, 정확한 레터링 값은 9-2절 참고). 사이드바 상단에 배치.
  2. **"CATEGORY" 내비게이션 목록** — 항목마다 라벨 + 우측 정렬 pill형 카운트 배지(radius 999). 활성 항목은 흰 배지, 비활성 항목은 앰버(`#FFCB47`) 배지 — 배경(teal)과 항상 반전 대비를 이루도록 한다(흰 배지는 어두운 텍스트, 앰버 배지도 어두운 텍스트).
  3. **"카테고리 관리"** — 각 카테고리 항목에 인라인 수정/삭제 소형 액션.
  4. **새 카테고리 추가 폼** — 입력창 1개 + 추가 버튼, 사이드바 최하단.
- 사이드바 안의 모든 항목은 브랜드색(teal) 배경 위이므로 텍스트는 항상 `text-primary`(ink `#1C1F21`) — 흰 텍스트 금지(AA 위반, Brand Guide 대비 규칙).

### 1-2. 본문 (테이블형 리스트, 고밀도)

- **컨테이너 padding**: 20px (여백을 넉넉히 주는 미니멀 스타일과 반대 방향 — 고밀도 정보 배치 우선).
- **블록 순서(위→아래)**, 블록 간 gap 14px:
  1. **상단바** — 화면 타이틀(좌) + 로그인 사용자명 + 로그아웃 버튼(우).
  2. **검색행** — 검색 입력창 + 검색 실행 버튼(coral, 파괴적 액션과 함께 coral을 쓰는 유일한 비파괴 예외).
  3. **"추가" 행** — 여러 입력 필드(이름/전화번호/주소/카테고리 등)를 하나의 그룹 컨테이너 박스로 묶고, 그 안에 추가 버튼(amber) 배치. 이 박스는 2.5px 잉크 보더 그룹 컨테이너로 테이블과 동일한 위계.
  4. **리스트 헤더** — 총 건수 표시(예: "총 24건").
  5. **테이블** — 헤더 행 + 데이터 행. 헤더는 앰버 틴트(`#FFF0CC`) 배경.
  6. **상태 메시지 배너** — 성공/오류/안내 메시지, 옅은 민트 틴트(`#E5F0ED`) 배경.

### 1-3. 빈 상태 (Contacts — Empty)

- 사이드바 구조는 동일하게 유지한다(내비게이션은 데이터 유무와 무관하게 항상 노출).
- 본문 테이블 헤더까지는 그대로 두되, 데이터 행 대신 **빈 상태 그래픽**을 배치한다. Brand Guide 일러스트레이션 방향에 따라 마스코트를 새로 그리지 않고 **로고 원형 심볼을 확대 재사용**한다(graphic-designer에게 새 그래픽 요청 불필요).
- 검색행/추가행은 데이터 유무와 무관하게 항상 노출(카테고리가 비어 있어도 새로 추가할 수 있어야 하므로).

## 2. 보더 위계 (Border Weight) — 그림자 전혀 쓰지 않음

elevation(그림자)은 이 레이아웃에서 전혀 사용하지 않는다. 구조적 중요도는 오직 잉크 보더(`#1A1A1A`) **두께**로만 표현한다.

| 두께 | 적용 대상 |
|---|---|
| 3px | 화면 최외곽 프레임 |
| 2.5px | 그룹 컨테이너 (테이블 전체, "추가" 행의 다중 입력 필드 묶음 박스) |
| 1.5px | 개별 입력창·버튼 |
| 1px | 행 내부 소형 액션 버튼(수정/삭제) |
| 1px, 색상 `#E0E0E0` (잉크 아님, 예외) | 테이블 행 사이 구분선(헤어라인) — 고밀도 리스트의 시각적 소음을 줄이기 위한 유일한 예외 |

이 5개 값(3px/2.5px/1.5px/1px잉크/1px헤어라인) 밖의 임의 보더 두께를 추가하지 않는다.

> **이 절은 B-2(62:6) 레이아웃 관찰 결과다.** 심볼·스와치 같은 "개별 유닛"에 적용되는 2px 잉크 아웃라인 규칙(브랜드 톤 원본 34:4 관찰)은 이 절과 출처·목적이 다르다 — 9절 안의 "주의 — 출처 구분" 박스를 함께 참고한다.

| 색상 | 용도 |

**텍스트 대비 절대 규칙**: 브랜드 3색(teal/coral/amber)은 전부 mid-tone이라 흰 텍스트와 페어링하면 AA 미달이다. 브랜드색 배경 위 텍스트는 항상 `text-primary`(ink `#1C1F21`) 고정 — 사이드바(teal 배경), amber 버튼, coral 버튼 모두 예외 없이 적용한다.

## 7. 장식 모티프 (Decorative Motif)

이 레이아웃 단계에서 반복되는 장식은 두 가지뿐이다:
1. **로고 원형 배지** — 사이드바 상단 1곳.
2. **pill형 카운트 배지**(radius 999) — 활성 행 흰 배지 / 비활성 행 앰버 배지, 배경과 항상 반전 대비.

기능 아이콘(검색/추가/수정/삭제/카테고리/로그아웃 등)은 graphic-designer가 이미 8종 제작 완료했다(`docs/design/graphic-assets.md` 참고, "Graphic Assets" 페이지 90:2, 노드 91:4~91:18). ui-designer는 이 원화를 임의로 새로 그리지 않고 그대로 가져다 조립한다.

## 8. ui-designer를 위한 안내

SCREENS 파일럿 3개(Login / Contacts — With Data / Contacts — Empty)를 제작할 때 이 문서를 **최우선 근거**로 삼는다. `docs/design/brand-guide.md`의 "레이아웃 관례 — B-2" 절보다 이 문서가 더 구체적인 실행 근거이며, 두 문서가 다르면 이 문서를 따른다. B-2를 "그대로 이어받아 발전(elevate)"시키는 것이 목표이며, 재해석·재설계가 아니다 — 위 수치·규칙에서 벗어난 임의의 spacing/radius/보더 값을 추가하지 않는다. design-systems 토큰은 아직 존재하지 않으므로(이 파일럿에서 사용된 값을 근거로 추후 추출됨), 이 문서와 Brand Guide의 원본 값을 직접 사용한다. **로고를 화면(사이드바 상단 등)에 배치할 때는 임의의 원+텍스트로 대체하지 말고, 반드시 아래 9-2절의 정확한 레터링 스펙(서체·크기·자간·색상·심볼 사양·간격·배치)을 그대로 재현한다** — 이전에 임의의 원+텍스트로 대체되어 재작업이 발생한 사례가 실제로 있었다.

## 9. 브랜드 퍼스낼리티 표현 장치 (Brand Personality Devices)

이 절은 **34:4(1차 브랜드 톤 원본 프레임)**를 brand-designer가 직접 다시 관찰해 기록한 내용이다(1~8절은 62:6 B-2 레이아웃 관찰 결과 — 관찰 대상 프레임이 다르다). spacing·radius·보더 두께 같은 "자로 잴 수 있는 것"은 위 1~6절에 이미 담겨 있고, 이 절은 그것만으로는 재현되지 않는 **컨셉의 성격**(네오팝 배지형 — "다정하지만 경계는 또렷하게")을 만드는 형태·색·조판 디테일을 다룬다. 아래 3가지는 서로 다른 관찰 대상이며 하나의 규칙("선을 두껍게 긋는다")으로 뭉뚱그리지 않는다.

### 9-1. 배지·스티커 형태 은유 (Badge/Sticker Motif)

- 34:4에서 반복되는 스티커 장치는 **2px 두께의 잉크 보더(`#1A1A1A`)**를 컬러 스와치(rounded-rect, radius 8px)와 원형 심볼에 동일하게 적용해 "테두리 있는 낱장 스티커"처럼 보이게 만든다.
- 그림자·그라디언트 없이 **플랫 솔리드 컬러만** 채워 스티커 특유의 평면 질감을 낸다.
- **적용 범위 규칙(중요)**: 이 아웃라인+플랫 처리는 **개별 유닛(스와치, 심볼) 단위에만** 적용되고, 이를 담는 **상위 컨테이너(워드마크 톤 카드, 페이지 배경)에는 적용되지 않는다.** 스티커 은유는 낱개 오브젝트 단위에서만 발생하고, 배경·카드 자체는 테두리 없이 열려 있다.

> **주의 — 출처 구분(2절 "보더 위계"와 혼동 금지)**: 이 절의 "2px 잉크 아웃라인은 개별 유닛에만 적용" 규칙은 34:4(브랜드 톤 원본)에서 관찰된 것이고, 위 2절의 3px/2.5px/1.5px/1px 규칙은 62:6(B-2 레이아웃)에서 관찰된 것이다. 2절은 화면 요소의 **구조적 중요도**(최외곽 프레임 > 그룹 컨테이너 > 개별 입력·버튼 > 소형 버튼)를 두께로 표현하는 규칙이고, 이 절은 심볼·스와치 같은 **낱개 오브젝트**에만 아웃라인을 적용하고 이를 담는 컨테이너에는 적용하지 않는다는 **"적용 대상의 범위"** 규칙이다. 두 규칙은 동시에, 서로 다른 레이어에서 적용된다 — 예: 사이드바 상단 로고 심볼은 이 절의 규칙(개별 유닛 2px 아웃라인)을 따르고, 그 심볼을 담는 사이드바 자체는 2절 규칙(사이드바는 컨테이너이므로 이 절의 아웃라인 대상이 아니며, 화면 최외곽 3px 규칙만 적용)을 따른다.

### 9-2. 정확한 워드마크 레터링 스펙 (Exact Wordmark Lettering Spec)

로고를 재현하는 워커는 아래 표의 값을 정확히 그대로 사용한다. 임의의 원+텍스트로 대체하지 않는다.

| 항목 | 값 |
|---|---|
| 워드마크 텍스트 | `yourbook.` — 전부 소문자 + 마침표로 끝남(대문자 없음). **서비스명 미확정 placeholder**(brand-guide.md 참고) — 임의로 최종 서비스명으로 바꾸지 않는다 |
| 서체 | Baloo 2 **Bold** (ExtraBold 아님 — 5절의 Display 웨이트 규칙과 별개) |
| 폰트 크기 | 26px (톤 카드 높이 79px 대비 약 33%) |
| 자간 | 0% (기본값) |
| 색상 | `#0F7A6E` (딥틸 — 주조색 `#17A398`보다 어둡게 톤다운, 밝은 원색 아님) |
| 심볼 크기 | 24×24 |
| 심볼 채움 | 코랄 `#FF5A76` |
| 심볼 스트로크 | 2px 잉크 `#1A1A1A`, **24px 이상 크기에서만 적용** (16px 파비콘은 스트로크 없이 채움만) |
| 심볼-워드마크 배치 | 가로 배치(심볼 좌, 워드마크 우) |
| 심볼-워드마크 간격 | 14px (심볼 지름 24px의 약 0.58배 비율 — 다른 스케일로 확대/축소해도 이 비율 유지) |
| 수직 정렬 | 심볼의 수직 중심 = 워드마크 텍스트 블록의 수직 중심 (오차 0.5px 이내로 거의 일치) |

**주의**: 기존 "로고/워드마크 사용 예시"(Figma `52:9`)의 스케일 예시는 이번 관찰과 다르게 워드마크를 Baloo 2 ExtraBold + ink 색상으로 잘못 표기했던 이력이 있다 — 이 표(Bold + `#0F7A6E`, 간격 14px)가 정정된 값이므로 이 표를 따른다.

### 9-3. 퍼스낼리티 이중성의 시각적 표현 (Warm-yet-Bounded Duality Devices)

"다정하지만 경계는 또렷하게"는 단일 규칙이 아니라 아래 3개의 장치가 겹쳐 만든다. 로고·장식 요소를 새로 만들거나 배치할 때 이 3가지를 함께 확인한다.

1. **선택적 아웃라인 위계**: 2px 잉크 보더는 "개인 소유" 낱개 유닛(스와치, 심볼)에만 적용되고 배경에는 적용되지 않아, 경계가 화면 전체가 아니라 개별 데이터 단위에서만 또렷해진다.
2. **오브젝트 내부의 색-형태 분리**: 심볼 하나에서 따뜻한 코랄 채움은 내부에, 단단한 잉크 링은 외곽에 두어 한 오브젝트 안에서 다정함과 경계를 동시에 표현한다.
3. **워드마크의 형태·색·구두점 삼중 대비**: 둥글고 캐주얼한 소문자 레터링(다정함) + 마침표라는 단호한 종결 부호(경계) + 채도를 낮춘 딥틸 색상(가벼운 원색이 아닌 신뢰감 있는 톤).

그림자·그라디언트가 전혀 없어 "다정함" 신호는 색·둥근 형태에서만, "경계" 신호는 보더 두께에서만 나오도록 일관된다 — 이 원칙은 B-2 레이아웃(2절, 그림자 없음·잉크 보더 위계)까지 그대로 이어진다. 새로운 장식 요소를 추가할 때 그림자·그라디언트를 넣지 않는다.
