# 모션 타이밍 & 퍼포먼스 가이드

이 문서는 interaction-designer(사용자 조작 반응 애니메이션)·motion-designer(독립 애니메이션 에셋)가 애니메이션을 만들 때 참조하는 **duration·easing·성능 표준**이자, design-qa가 감사할 때 쓰는 **검증 기준**이다. `docs/harness/design-team/figma-page-format-guide.md` 등과 같은 성격의 포터블 참조 문서로, 프로젝트 고유 값 없이 일반 원칙만 담는다.

**이 문서가 다루지 않는 것**: 어떤 상태(Hover/Press 등)가 필요한지는 `component-state-guide.md`를 따른다. 이 문서는 오직 "그 상태 전환·애니메이션이 얼마나 오래, 어떻게 움직여야 하는지"만 다룬다.

## 1. Duration 기준

- 마이크로 인터랙션(호버, 클릭 피드백): 100~200ms
- 화면 전환: 200~400ms
- 로딩 스피너/프로그레스바 등 반복 애니메이션: 800~1200ms 주기(그보다 빠르면 산만하고, 느리면 멈춘 것처럼 보인다)

## 2. Easing

급발진/급정지 없이 자연스럽게 — ease-in-out 계열을 기본으로 삼는다.

## 3. 전환 애니메이션은 정보다

모든 전환은 "무엇이 어디서 왔고 어디로 가는지"(공간적 연속성) 또는 "지금 무슨 일이 일어나고 있는지"(상태 피드백)를 설명해야 한다. 설명하지 못하는 움직임은 뺀다.

## 4. 반복 애니메이션은 루프 가능해야 한다

로딩 인디케이터, 애니메이션 아이콘처럼 반복되는 애니메이션은 끊김 없이 반복되는지(마지막 프레임과 첫 프레임이 자연스럽게 이어지는지) 확인한다.

## 5. 퍼포먼스

transform/opacity 위주로 설계해 60fps를 유지할 수 있는 방식으로 만든다 — width/height/색상 직접 애니메이션처럼 리플로우를 유발하는 속성 위주 설계를 피한다.

## 체크리스트

- [ ] 마이크로 인터랙션이 100~200ms 범위인가?
- [ ] 화면 전환이 200~400ms 범위인가?
- [ ] 반복 애니메이션이 800~1200ms 주기이고 끊김 없이 루프되는가?
- [ ] easing이 급발진/급정지 없는 ease-in-out 계열인가?
- [ ] 이 전환/애니메이션이 설명하는 정보(공간적 연속성 또는 상태 피드백)가 명확한가?
- [ ] transform/opacity 위주로 설계돼 60fps를 해치는 속성(레이아웃 리플로우 유발)을 안 쓰는가?
