# 아이콘 크래프트 가이드 — Basic/Visual 트랙 구분

이 문서는 graphic-designer가 아이콘을 그릴 때 참조하는 **크래프트 표준**이자, design-qa가 완성된 아이콘 컴포넌트를 감사할 때 쓰는 **검증 기준**이다. `docs/harness/design-team/figma-page-format-guide.md`가 "문서화 페이지의 서식"을 다루는 것과 같은 성격의 포터블 참조 문서로, **구체적 hex 값이나 노드 ID를 담지 않고 일반 원칙만** 담는다. 실제 프로젝트가 채택한 스트로크 두께 수치, 브랜드 컬러 값 등은 항상 `docs/design/brand-guide.md`·`docs/design/design-system.md`(프로젝트별 산출물)를 참고한다.

**이 문서가 다루지 않는 것**: 어떤 아이콘을 그릴지, 어떤 스타일 방향(라인/플랫/3D 등)을 쓸지는 다루지 않는다 — 그건 brand-designer가 Brand Guide에서 정한다. 이 문서는 오직 "이미 정해진 스타일 방향 안에서, 아이콘을 두 트랙 중 어느 쪽 크래프트 규칙으로 그릴지"만 다룬다.

## 왜 두 트랙으로 나누는가

아이콘 세트 전체를 하나의 두께·면 처리로 통일해버리면 두 가지 문제가 동시에 생긴다.

- 화면 어디서나 자주 반복되는 사소한 액션(닫기, 화살표 등)까지 브랜드의 굵은 아웃라인을 쓰면 화면이 과하게 투박하고 무거워 보인다.
- 반대로 브랜드 퍼스낼리티를 실제로 전달해야 하는 아이콘(카테고리, 알림 등)까지 가늘고 중립적인 선으로 그리면 그 아이콘이 있으나 마나 한 존재감을 가진다.

그래서 아이콘을 **기본(Basic/Utility)**과 **비주얼(Visual/Feature)** 두 트랙으로 나누고, 트랙마다 다른 크래프트 규칙을 적용한다.

## 트랙 판정 기준

새 아이콘을 그리기 전에, 또는 감사 중 이미 그려진 아이콘의 트랙이 맞는지 확인할 때 아래 질문으로 판정한다.

- **이 아이콘이 화면 여러 곳에서 반복적으로 등장하고, 그 자체로 브랜드를 표현할 필요가 없는가?** (예: 화면 전환 화살표, 닫기, 펼치기/접기, 로그아웃처럼 기능이 명확하고 흔한 UI 관례를 따르는 아이콘) → **기본(Basic/Utility) 트랙**
- **이 아이콘이 브랜드 퍼스낼리티를 실제로 전달해야 하는 자리에 쓰이는가?** (예: 카테고리 구분, 알림, 빈 상태 그래픽처럼 사용자가 그 화면/기능의 "성격"을 느끼는 지점) → **비주얼(Visual/Feature) 트랙**

두 조건에 걸쳐 있어 보이면, "이 아이콘이 없어져도 화면의 브랜드 인상이 달라지지 않는가"를 기준으로 삼는다 — 달라지지 않으면 기본 트랙이다.

## 트랙별 크래프트 규칙

- **기본(Basic/Utility) 트랙**
  - 얇고 중립적인 스트로크만 사용한다 — 브랜드의 굵은 잉크 아웃라인보다 눈에 띄게 가는 두께로, 화면에서 튀지 않고 "배경 소음"처럼 자연스럽게 녹아들어야 한다.
  - **면색(fill)을 넣지 않는다 — 순수 스트로크(outline)만 사용한다.** 브랜드색이든 다른 색이든 면을 채우지 않는다.
- **비주얼(Visual/Feature) 트랙**
  - Brand Guide가 정한 굵은 잉크 아웃라인 + 브랜드색 평면 채색을 그대로 유지해 브랜드 특징을 살린다. 정확한 두께·색 값은 `docs/design/brand-guide.md`를 참고한다.

## 세트 내 일관성

같은 트랙끼리는(기본 아이콘끼리, 비주얼 아이콘끼리) 스트로크 두께와 모서리 반경(corner radius)을 동일하게 유지한다. 트랙 안에서 두께가 들쭉날쭉하면 세트 전체가 만든 사람이 다른 것처럼 보인다.
