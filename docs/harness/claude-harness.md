# Claude Code 하네스 구조 — 어떤 파일이 무엇을 통제하는가

이 문서는 이 프로젝트에서 Claude(메인 세션)와 그 하위 서브에이전트들의 행동을 어떤 파일이 통제하는지 정리한다. 새 프로젝트에서 하네스를 재구성할 때 이 구조를 그대로 참고하면 된다.

## 1. 메인 세션(Claude)을 직접 통제하는 파일

| 파일 | 위치 | 역할 |
|---|---|---|
| `CLAUDE.md` | 프로젝트 루트 (git 추적) | 이 프로젝트 전용 지침 — 코딩 규칙, 검증 명령, 금지사항(원격 push 승인 필요 등), 행동 지침 참조. 프로젝트를 열 때마다 자동으로 읽힌다. |
| `docs/karpathy_skills.md` | 프로젝트 (git 추적) | `CLAUDE.md`가 `@`로 참조하는 행동 원칙 문서(Think Before Coding, Simplicity First 등). |
| `~/.claude/projects/{프로젝트경로 슬러그}/memory/*.md` + `MEMORY.md` | 사용자 홈 디렉토리 (git 밖, 이 프로젝트 경로에만 묶임) | 세션이 바뀌어도 남는 지속 메모리 — 사용자 피드백, 프로젝트 배경, 참고 위치 등. `MEMORY.md`가 인덱스 역할. **다른 프로젝트로 자동으로 안 옮겨진다** — 프로젝트 절대경로가 키다. |

## 2. 서브에이전트(팀원)를 통제하는 파일 — 각자 호출될 때 자기 파일을 읽는다

| 파일 | 위치 | 역할 |
|---|---|---|
| `.claude/agents/*.md` | 프로젝트 (git 추적) | 이 프로젝트 전용 에이전트 정의(이름/설명/도구/모델 + 역할·판단기준·하지 말 것). 존재하면 아래 전역 버전보다 **우선 적용**된다. |
| `~/.claude/agents/*.md` | 사용자 전역 (git 밖) | 모든 프로젝트에서 기본으로 쓰이는 에이전트 정의 — 새 프로젝트를 열었을 때 프로젝트 로컬 버전이 없으면 이게 쓰인다. 이 디자인팀(design-pl 이하 12개)은 여기 복사돼 있어 다른 프로젝트에서도 바로 쓸 수 있다. |
| `.claude/skills/*/SKILL.md` | 프로젝트 (git 추적) | Skill 도구로 로드하는 절차서(예: `design-concept-round`). |
| `~/.claude/skills/*/SKILL.md` | 사용자 전역 | 위와 동일하되 모든 프로젝트에서 기본 제공. |
| `.claude/agent-memory/{에이전트명}.md` | 프로젝트 (git 추적) | 각 에이전트의 **최근 작업 로그만**(휘발성, 5개 캡 — git 히스토리가 전체 이력의 원본). 브랜드 결정사항·토큰 인벤토리 같은 지속 상태는 여기 두지 않는다(3번 참고). |

## 3. 재사용 가능한 것 vs 이 프로젝트만의 데이터 — 디렉토리로 분리했다

- **`docs/harness/`(재사용 가능, 새 프로젝트에 그대로 복사)**: 이 문서(`claude-harness.md`)와 `design-team/figma-file-organization.md` — 절차/구조만 담겨 있고 프로젝트 고유 데이터(문서 번호·색상값·노드 ID)는 하드코딩돼 있지 않다. 앞으로 다른 팀(예: 기획팀)의 절차서도 `docs/harness/{팀명}/` 형태로 여기 추가한다. `.claude/agents/*.md`, `.claude/skills/*`도 같은 성격(전역 위치에도 복사돼 있음, 2번 참고).
- **`docs/design/`(이 프로젝트만의 데이터, 다른 프로젝트로 그대로 못 옮김)**: `brand-guide.md`, `graphic-assets.md`, `design-system.md`, `confirmed/*.md` — 실제 브랜드 컬러(Warm Ledger), 레이아웃 확정 스펙(B-2) 등 이 프로젝트에서 실제로 결정된 내용. 새 프로젝트에서는 `docs/harness/design-team/figma-file-organization.md`를 템플릿 삼아 새 프로젝트의 `docs/design/`에 이런 문서들을 새로 채워야 한다.

## 4. 권한/설정 파일 (행동 규칙이 아니라 "무엇을 할 수 있는지")

| 파일 | 역할 |
|---|---|
| `.claude/settings.json` / `.claude/settings.local.json` | 도구 권한, hook 설정 등. |

## 5. 우선순위 요약

프로젝트 로컬 파일(`.claude/agents/`, `.claude/skills/`)이 항상 전역(`~/.claude/agents/`, `~/.claude/skills/`)보다 우선한다. 전역은 "이 프로젝트에 아직 로컬 버전이 없을 때의 기본값"이다. 에이전트 파일을 프로젝트에서 수정하면, 전역에도 같은 이름 파일이 있는지 확인해 함께 갱신해야 여러 프로젝트에서 일관성이 유지된다(수동 동기화 — 자동 동기화 기능은 없음).
